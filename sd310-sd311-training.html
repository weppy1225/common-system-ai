핵심 원칙 — 한 방향 흐름

요건(ui.md)이 단일 진실원천(SoT)입니다. 화면(HTML)은 ui.md 에서 파생됩니다. 화면을 바꾸고 싶으면 ui.md 를 먼저 고치고 SD_311 을 다시 돌립니다.

왜 둘로 나눴나

  • 요건 정리(사람 판단)와 화면 생성(기계 변환)을 분리
  • 요건 MD는 사람이 검토·수정 가능한 텍스트
  • 같은 ui.md 로 PC(SD_311)·PDA(SD_312) 화면 모두 생성

생성 원칙 — 공통 CSS/JS·템플릿은 절대 수정하지 않습니다. 화면은 공통 자산을 <link>/<script> 로 참조하고, 정해진 템플릿의 플레이스홀더만 채웁니다.

건드리지 않는 공통 자산

파일역할
prototype/_common/wms-ui.css공통 레이아웃·컴포넌트 CSS (재정의 금지)
prototype/_common/wms-common.js공통 팝업·페이징·수평스크롤 유틸
prototype/_common/_template/base.html전체 레이아웃 뼈대
_template/grid1.html · grid2-tb/lr · grid3-lrt/tlbUI유형별 메인 영역 템플릿 (1개 선택)
생성 HTML에 반드시 포함: 
<link rel="stylesheet" href="../_common/wms-ui.css">
<script src="../_common/wms-common.js"></script>

SD_311 절대 금지 — 자주 틀리는 지점

  • window.open() 금지 → 공통 openCpPopup(id) / openPdPopup(id) 사용
  • 인라인 style 로 overflow/width/height 직접 지정 금지 → 공통 클래스 사용
  • .panel .toolbar .modal .btn 등 wms-ui.css 공통 CSS 재정의 금지
  • ui.md 에 없는 버튼(엑셀등록/다운로드 등) 임의 추가 금지
  • 입력 필드 placeholder 임의 추가 금지 · 빈 stub 함수 금지
  • icon-preview.html 에 없는 아이콘 사용 금지

3 mdpc01 실습 — 처음부터 끝까지

왜 mdpc01 인가 — PC관리는 index.html 메뉴 등록도, spec/mdpc01/ · prototype/mdpc01/ 산출물도 아무것도 없는 완전 신규 상태입니다. 메뉴 등록(SD_311 5단계)까지 포함해 두 스킬을 처음부터 끝까지 시연하기에 가장 적합합니다.

실습 순서

# 1단계 — 화면요건 작성 (6단계 인터뷰)
/SD_310_UI mdpc01
   → 질문에 답변 → 요약 확인 → spec/mdpc01/mdpc01-02-ui.md 생성

# 2단계 — 프로토타입 생성
/SD_311 mdpc01
   → ui.md 읽기 → mock-data.js + wireframe.html 생성 + 메뉴 등록

# 3단계 — 브라우저 확인
prototype/index.html 열기 → 좌측 [기준정보] → [PC관리] 클릭

생성 전·후 디렉토리 비교

실습 전 — 완전 신규

spec/ (mdpc01 폴더 없음) prototype/ ├── index.html ← PC관리 메뉴 없음 └── _common/ (mdpc01 폴더 없음)

실습 후

spec/ └── mdpc01/ └── mdpc01-02-ui.md SD_310 prototype/ ├── index.html 메뉴 추가(신규) ├── _common/left-menu.html 메뉴 추가(신규) └── mdpc01/ ├── mdpc01-wireframe.html SD_311 └── mdpc01-mock-data.js SD_311

메뉴 등록 결과 (SD_311 5단계가 신규 추가하는 라인)

<li class="submenu-item"
    onclick="loadContent('mdpc01/mdpc01-wireframe.html', 'PC관리')">PC관리</li>

PC관리 메뉴는 처음엔 없으므로, SD_311 이 기준정보 그룹 submenu에 위 라인을 신규로 추가합니다 (index.html + left-menu.html 양쪽). 클릭 시 새로 생성된 wireframe 이 iframe 으로 로드됩니다.
