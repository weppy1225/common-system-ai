# MDWH01 기본설계 — 창고

## 1. 업무 정의

창고(MDWH01)는 물류센터 내에 존재하는 물리적 또는 논리적 공간 단위를 등록하고 관리하는 기준정보 메뉴다. 창고는 반드시 하나의 물류센터에 속하며, 창고그룹(예: 입고그룹·반품그룹·제품그룹 등)으로 분류된다. 각 창고는 입고·출고·피킹·반품·예외 등 하나 이상의 처리기능을 보유해야 하며, 이 정보는 이후 입출고, 재고 이동, 재고 수불 등 모든 물류 업무의 기준이 된다.

창고를 새로 등록하면 시스템은 해당 창고에 기본 위치(로케이션)를 자동으로 생성하고 바코드를 부여한다. 또한 창고가 위탁(물류대행) 센터에 속할 경우, 해당 센터를 위탁 중인 모든 사업장과 창고를 자동으로 매핑한다.

창고 정보는 재고 수불 이력이 존재하는 경우 삭제가 불가능하다. 재고 이력이 있는 창고는 사용 중지 처리로만 운영을 종료할 수 있다.

## 2. 관리 대상 정보

| 항목 | 설명 |
|---|---|
| 물류센터 | 창고가 속한 물류센터 (필수, 등록 후 변경 불가) |
| 창고그룹 | 창고의 업무 유형 분류 (입고그룹/반품그룹/제품그룹/출고그룹/외주그룹/예외그룹 등, 등록 후 변경 불가) |
| 창고명 | 창고의 명칭 (필수, 동일 센터 내 중복 불가) |
| IF창고ID | 외부 시스템(ERP) 연동용 창고 식별자 |
| 가용재고 여부 | 이 창고의 재고를 주문 가용 재고로 집계할지 여부 |
| 온도구분 | 창고의 온도 관리 환경 (상온/냉장/냉동 등) |
| 처리기능 | 창고에서 수행 가능한 업무 종류 (입고·출고·피킹·반품·예외·세트작업·전환) |
| 사용여부 | 창고의 활성/비활성 상태 (수정 모드에서만 변경 가능) |
| 기본창고 여부 | 센터의 기본 창고로 지정된 창고인지 여부 (목록 표시용, 기본창고는 삭제 불가) |

## 3. 업무 참여자/역할

| 역할 | 업무 범위 |
|---|---|
| 운영 담당자 (등록 사업장) | 자신이 속한 사업장 기준으로 창고 등록·수정·삭제 가능 |
| 위탁 사업장 담당자 | 물류대행 센터의 창고를 조회만 가능 (등록·수정·삭제 불가) |
| 시스템 | 창고 등록 시 기본 위치 자동 생성, 위탁 사업장-창고 자동 매핑 |

## 4. 업무 흐름 전체

```
[사용자]
   │
   ├─ 조회 조건 입력 (물류센터, 창고그룹, 창고명, 사용여부)
   │         │
   │         ▼
   │   [창고 목록 조회]
   │         │
   │   ┌─────┴──────────────────────────────┐
   │   │                                    │
   ▼   ▼                                    ▼
[신규 등록]                             [기존 창고 선택]
   │                                        │
   │                               ┌────────┴────────┐
   │                               │                 │
   │                         [수정 팝업 열기]    [삭제 요청]
   │                               │                 │
   ▼                               ▼                 ▼
[등록 팝업 입력]           [정보 수정 후 저장]   [삭제 가능 여부 확인]
   │                               │                 │
   ▼                               ▼        ┌────────┴────────┐
[유효성 검증]               [유효성 검증]    │                 │
   │                               │     [재고 이력 없음]  [재고 이력 있음]
   ▼                               ▼        │                 │
[창고 저장]                 [창고 정보 갱신]  ▼            [삭제 불가 안내]
   │                                    [삭제 완료]
   ▼
[기본 위치 자동 생성]
   │
   ▼
[위탁 사업장 자동 매핑]
   │
   ▼
[목록 갱신]
```

## 5. 세부 업무 시나리오

### 5-1. 창고 신규 등록

1. 담당자가 목록 화면에서 추가 버튼을 클릭한다.
2. 등록 팝업에서 물류센터, 창고그룹, 창고명, 온도구분을 입력한다.
3. 창고그룹을 선택하면 해당 그룹의 기본 처리기능이 자동으로 설정되며, 일부 기능은 고정되어 변경할 수 없다.
4. 등록 버튼 클릭 시 시스템이 아래를 검증한다.
   - 창고그룹과 창고명 필수 여부
   - 동일 센터 내 창고명 중복 여부
   - 처리기능 최소 1개 이상 선택 여부
5. 검증 통과 후 창고를 저장하고, 기본 위치를 자동 생성한다.
6. 창고를 위탁 센터에 등록한 경우, 해당 센터를 사용하는 모든 위탁 사업장과 창고를 자동으로 매핑한다.

### 5-2. 창고 정보 수정

1. 담당자가 목록에서 창고를 선택하고 수정 버튼(또는 더블클릭)을 클릭한다.
   - 위탁 사업장 담당자는 수정 팝업을 열 수 없다.
2. 수정 팝업에서 변경 가능한 항목(창고명, IF창고ID, 처리기능, 사용여부 등)을 수정한다.
   - 물류센터, 창고그룹은 등록 후 변경 불가.
   - 창고그룹에 따라 수정 가능한 처리기능 항목이 제한된다.
3. 저장 버튼 클릭 시 아래를 검증한다.
   - 창고명 중복 여부 (자기 자신 제외)
   - 처리기능 최소 1개 이상
   - 사용 중지 처리 또는 처리기능 변경 시, 해당 기능을 가진 다른 창고가 센터 내에 최소 1개 이상 존재하는지 확인 (입고·출고(fr)·출하(fr)·반품·예외 5가지 필수 보장)
4. 수정 저장 시 해당 창고의 기본 위치명도 함께 갱신된다.
5. IF창고ID를 입력하면 사업장-창고 매핑 테이블의 연동 식별자도 함께 갱신된다.

### 5-3. 창고 삭제

1. 담당자가 목록에서 삭제할 창고를 하나 이상 선택하고 삭제 버튼을 클릭한다.
2. 기본창고로 지정된 창고는 선택해도 삭제 요청 자체가 차단된다 (화면 단에서 먼저 검증).
3. 시스템이 삭제 가능 여부를 검증한다.
   - 재고 수불 이력(입출고 기록)에 해당 창고가 사용된 경우 삭제 불가.
   - 삭제 후 각 처리기능(입고·피킹·출고·반품·예외)을 보유한 창고가 센터 내에 최소 1개 이상 남아야 한다.
4. 검증 통과 시 창고, 해당 창고의 위치, 사업장-창고 매핑을 모두 삭제한다.

## 6. 상태 흐름

창고는 사용여부 단일 상태로 관리된다.

```
          [등록]
            │
            ▼
         [사용]
          (use_yn=Y)
            │
            │ ← 사용 중지 수정 (단, 해당 기능을 가진 다른 창고가 존재해야 함)
            ▼
         [미사용]
          (use_yn=N)
            │
            │ ← 다시 사용으로 수정 가능
            ▼
         [사용]
```

| 상태 | 설명 | 전이 조건 |
|---|---|---|
| 사용 | 창고가 정상 운영 중 | 등록 시 기본값 |
| 미사용 | 창고 운영 중단 | 해당 처리기능을 가진 다른 창고가 최소 1개 존재해야 전환 가능 |

## 7. 핵심 업무 규칙

**BR-1. 동일 센터 내 창고명 중복 불가**
같은 물류센터 안에서 동일한 창고명을 가진 창고는 2개 이상 존재할 수 없다. 등록 및 수정 시 모두 적용된다.

**BR-2. 처리기능 최소 1개 이상 필수**
창고는 반드시 입고·피킹·출고·반품·예외·세트작업·전환 중 하나 이상의 처리기능을 보유해야 한다. 모든 기능이 비선택 상태로는 저장할 수 없다.

**BR-3. 각 처리기능별 최소 1개 창고 보장 (입고·피킹·출고·반품·예외)**
삭제 또는 기능 변경(사용 중지 포함) 시, 해당 처리기능을 담당하는 창고가 센터 내에 최소 1개 이상 남아야 한다. 해당 기능을 가진 마지막 창고를 삭제하거나 기능을 제거할 수 없다.

**BR-4. 재고 이력이 있는 창고는 삭제 불가**
재고 수불 기록(입출고 이력)에 등장한 창고는 물리 삭제가 불가능하다. 담당자는 사용 중지 처리로 운영을 종료해야 한다.

**BR-5. 기본창고는 삭제 불가**
센터의 기본창고로 지정된 창고는 화면에서 삭제 요청 자체를 차단한다.

**BR-6. 창고그룹에 따른 처리기능 자동 설정 및 제한**
창고그룹 선택 시 기본 처리기능이 자동으로 설정된다. 창고그룹에 따라 일부 처리기능은 변경이 불가능하다.

| 창고그룹 | 자동 설정 처리기능 | 변경 불가 항목 |
|---|---|---|
| 입고그룹(G10) | 입고 | 입고, 반품, 예외, 세트, 전환 |
| 반품그룹(G20) | 반품 | 모든 기능, 가용재고 |
| 제품그룹(G30) | 출고(fr) | 반품, 출고(fr), 사용여부 |
| 출고그룹(G50) | 출하(fr) | 모든 기능, 사용여부, 가용재고 |
| 외주그룹(G70) | 입고·반품·출하·예외·세트·전환 | 입고, 출고(fr), 출하(fr), 가용재고 |
| 예외그룹(G90) | 예외·세트 | 모든 기능, 사용여부, 가용재고 |

**BR-7. 위탁 센터 창고 등록 시 위탁 사업장 자동 매핑**
창고가 속한 센터를 물류대행으로 이용 중인 사업장이 있으면, 창고 등록 시 해당 사업장과 창고가 자동으로 매핑된다.

**BR-8. 위탁 사업장은 창고 수정 불가**
물류대행 센터 소속 창고는 위탁 사업장 담당자가 등록·수정·삭제할 수 없다. 조회만 가능하다.

**BR-9. 등록 시 기본 위치 자동 생성**
창고를 신규 등록하면 "BL" 코드의 기본 위치가 자동으로 생성되고, 창고그룹 코드를 포함한 바코드가 부여된다. 창고명 변경 시 기본 위치명도 함께 갱신된다.

## 8. 권한별 업무 범위

| 구분 | 조회 | 등록 | 수정 | 삭제 |
|---|---|---|---|---|
| 등록 사업장 담당자 | 가능 | 가능 | 가능 | 가능 |
| 위탁 사업장 담당자 | 가능 | 불가 | 불가 | 불가 |

- 조회 시 자신의 사업장에 연결된 창고만 표시된다.
- 위탁 사업장 담당자에게는 추가·수정·삭제 버튼이 숨김 처리된다.
- 위탁 사업장 담당자가 더블클릭으로 수정 팝업을 시도해도 열리지 않는다.

## 9. 한 줄 요약

> 창고 메뉴는 물류센터 내 창고의 처리기능과 위탁 사업장 매핑을 관리하며, 재고 이력이 없고 기능 최소 보장 조건을 충족하는 경우에만 삭제가 허용된다.
