# MDBZ01 데이터 모델 (테이블·상태값)

## 1. 물리 테이블 목록

| 업무 개념      | 물리 테이블명         | 업무 역할                                             |
| ---------- | --------------- | ------------------------------------------------- |
| 사업장        | MDM_BIZ         | 사업장의 기본정보(사업장명, 사업자번호, 주소, 로고 등)를 저장하는 핵심 마스터 테이블 |
| 사업장-사업장 관계 | MDM_BIZ_BIZ     | 두 사업장 간의 연계 관계(예: 의뢰사업장-대행사업장)를 관리하는 관계 테이블       |
| 사용자-사업장 권한 | MDM_USER_BIZ    | 사용자가 접근 가능한 사업장 목록을 관리하는 권한 테이블                   |
| 문서번호       | MDM_DOC_NO      | 사업장별 문서번호 채번 규칙을 저장하는 테이블 (사업장 신규 생성 시 기본값 복제)    |
| 물류센터       | MDM_CENTER      | 물류센터의 기본정보(센터명, 주소, 물류대행 여부 등)를 저장하는 센터 마스터 테이블   |
| 사업장-센터 관계  | MDM_BIZ_CENTER  | 사업장과 센터 간의 관계(자사 운영/위탁 운영)와 승인 상태를 관리하는 관계 테이블    |
| 사용자-센터 권한  | MDM_USER_CENTER | 사용자가 접근 가능한 센터 목록을 관리하는 권한 테이블                    |
| 사용자        | MDM_USER        | 사용자 기본정보 및 권한 유형을 저장하는 사용자 마스터 테이블                |
| 창고         | MDM_WH          | 센터 내 창고 정보를 저장하는 창고 마스터 테이블 (센터 생성 시 기본 창고 자동 생성) |
| 사업장-창고 관계  | MDM_BIZ_WH      | 사업장이 접근 가능한 창고 목록을 관리하는 관계 테이블                    |
| 위치(로케이션)   | MDM_LOC         | 창고 내 위치 정보를 저장하는 테이블 (센터 생성 시 기본 위치 자동 생성)        |
| 파일         | SM_FILE         | 업로드된 파일 정보(로고 이미지 등)를 저장하는 공통 파일 테이블              |

## 2. 상태값 / 코드 규칙

### 2-1. 사업장-센터 위탁 관계 상태 (MDM_BIZ_CENTER)

사업장과 센터 간의 관계는 두 개의 컬럼(승인 여부 + 사용 여부)의 조합으로 상태를 표현한다.

| 상태명 | 승인 여부 | 사용 여부 | 의미 |
|---|---|---|---|
| REQUEST (신청 중) | N | N | 의뢰를 보냈으나 아직 대행업체가 응답하지 않은 상태 |
| ACCEPT (승인 완료) | Y | Y | 대행업체가 의뢰를 수락한 상태 |
| DENIED (거절됨) | Y | N | 대행업체가 의뢰를 거절한 상태 |

```
상태 전이도:
  (데이터 없음: 신청 가능)
          │
          │ 위탁 의뢰 신청
          ▼
  REQUEST (승인여부=N, 사용여부=N)
          │
          ├── 수락 ──►  ACCEPT (승인여부=Y, 사용여부=Y)
          │
          ├── 거절 ──►  DENIED (승인여부=Y, 사용여부=N)
          │                  │
          │                  └── 재신청 → REQUEST 로 초기화
          │
          └── 취소 ──►  (데이터 삭제: 신청 가능으로 복귀)
```

### 2-2. 사용 여부 코드 (공통)

모든 테이블에서 사용 여부를 Y/N 단일 문자 코드로 표현한다.

| 코드값 | 의미 |
|---|---|
| Y | 사용 중 |
| N | 미사용 (비활성 또는 소프트 삭제 상태) |

### 2-3. 사업장 구분 코드 (MDM_BIZ.biz_div_cd)

| 코드값 | 의미 | 비고 |
|---|---|---|
| OWN | 자사물류 | 직접 물류를 운영하는 사업장 |
| TPL | 물류대행 | 외부 물류대행사에 창고 운영을 위탁하는 사업장 |
| SHIPPER | 화주 | 배송을 의뢰하는 고객사. 수정 가능 사업장 목록에서 제외됨 |

### 2-4. 센터 물류대행 여부 (MDM_CENTER.tpl_yn)

| 코드값 | 의미 |
|---|---|
| Y | 물류대행 서비스 제공 가능한 센터 |
| N | 물류대행 서비스 제공 불가 센터 |

### 2-5. 사용자 권한 유형 코드 (MDM_USER.auth_type_cd)

MDM_BIZ_CENTER.editableYn 계산 및 searchEditableBizs 쿼리 분기에서 확인되는 코드값.

| 코드값 | 의미 | 사업장 접근 범위 |
|---|---|---|
| SUPER | 슈퍼 권한 | 조건 없이 수정 가능 사업장 전체 |
| BIZ | 사업장 권한 | MDM_USER_BIZ에 등록된 사업장만 |
| CENTER | 센터 권한 | MDM_USER_CENTER에 등록된 센터의 사업장만 |

### 2-6. 위탁 의뢰 검색 결과 상태 표시 (searchTplBizCenter SQL CASE 결과)

물류대행업체 검색 팝업에서 각 센터의 현재 의뢰 상태를 문자열로 표시한다.

| 표시값 | 조건 | 의미 |
|---|---|---|
| 승인 | 승인여부=Y, 사용여부=Y | 이미 수락된 센터 |
| 거절 | 승인여부=Y, 사용여부=N | 거절된 센터 (재신청 가능) |
| 신청중 | 승인여부=N, 사용여부=N | 신청 후 응답 대기 중 |
| 신청가능 | 데이터 없음 | 아직 의뢰하지 않은 센터 |
