# MDBZ01 기본설계 — 사업장

## 1. 업무 정의

사업장 화면은 WMS 시스템에 등록된 사업장의 기본 정보(사업자등록번호, 대표자, 주소, 로고 등)와 해당 사업장이 운영하는 물류센터 목록을 관리하는 기준정보 화면이다. 사업장 담당자가 자신이 속한 사업장의 정보를 열람하고 수정할 수 있으며, 물류센터를 추가·수정·삭제할 수 있다.

시스템에 가입한 사업장은 직접 물류를 운영하는 자사물류 형태와, 외부 물류대행사에 창고 운영을 위탁하는 물류대행 형태로 구분된다. 물류대행 방식의 경우 대행업체와의 위탁 계약 의뢰·승인 프로세스가 별도로 존재한다. 다만 현재 이 위탁 관련 일부 기능은 사업상 이유로 비활성화(주석 처리)되어 있다.

사업장 관리자는 수정 가능한 사업장 목록을 조회한 후 대상 사업장을 선택하고 정보를 변경한다. 이때 조회 가능한 사업장의 범위는 로그인 사용자의 권한 등급에 따라 달라진다.

## 3. 업무 참여자/역할

| 참여자 | 역할 |
|---|---|
| 사업장 관리자 | 사업장 기본정보 수정, 물류센터 추가·수정·삭제 수행 |
| 슈퍼 권한 사용자 | 수정 가능한 전체 사업장에 접근 가능. 신규 센터 등록 시 자동으로 센터 접근 권한 부여됨 |
| 사업장 권한 사용자 | 자신이 속한 사업장만 접근 가능 |
| 센터 권한 사용자 | 자신이 속한 센터의 사업장만 접근 가능 |
| 물류대행업체 담당자 | 위탁 의뢰를 수락 또는 거절하는 역할 (위탁 기능 관련) |

## 5. 세부 업무 시나리오

### 시나리오 A: 사업장 정보 수정

1. 사용자가 화면에 진입하면 수정 가능한 사업장 목록이 드롭다운으로 표시된다.
2. 첫 번째 사업장이 기본 선택되며, 선택된 사업장의 기본정보와 센터 목록이 화면에 로드된다.
3. 사용자가 사업장명(필수), 대표자, 주소 등 항목을 수정하고 저장 버튼을 클릭한다.
4. 사업장명이 비어 있는 경우 경고 메시지가 표시되고 저장이 거부된다.
5. 로고 이미지를 새로 업로드하거나 기존 이미지를 삭제할 수 있다.
6. 저장 성공 시 화면이 갱신되고 헤더의 사업장명도 즉시 업데이트된다.

### 시나리오 B: 물류센터 추가

1. 사용자가 센터 그리드에서 행 추가를 선택한다.
2. 새 행에 센터명(필수)과 기타 정보를 입력한다.
3. 저장 버튼을 클릭하면 입력 유효성이 검증된다.
4. 센터명이 비어 있으면 경고 메시지 표시 후 중단된다.
5. 센터명이 동일 사업장 내에서 중복되면 오류 메시지가 표시된다.
6. 등록이 완료되면 해당 센터에 대한 기본 창고가 자동으로 생성된다 (템플릿 기반).
7. 슈퍼 권한을 보유한 모든 사용자에게 새 센터의 접근 권한이 자동 부여된다.

### 시나리오 C: 물류센터 수정

1. 사용자가 그리드에서 기존 센터 행을 선택하여 내용을 수정한다.
2. 위탁(물류대행)으로 운영 중인 센터는 행을 직접 편집할 수 없다.
3. 사용 중지로 변경하거나 물류대행 여부를 해제하려면, 해당 센터에 활성화된 위탁 계약이 없어야 한다. 위탁 계약이 존재하면 오류 메시지가 표시된다.
4. 저장 버튼 클릭 시 변경된 항목(추가·수정·삭제)이 일괄 처리된다.
5. 저장 후 사업장 내 사용 중인 센터가 하나도 없는 경우 오류 메시지가 표시되고 저장이 거부된다.

### 시나리오 D: 물류센터 삭제

1. 그리드에서 행을 삭제 표시한 후 저장 버튼을 클릭한다.
2. 삭제하려는 센터에 권한이 부여된 일반 사용자(슈퍼 권한 제외)가 존재하면 삭제가 거부된다.
3. 삭제하려는 센터에 위탁 계약이 존재하면 삭제가 거부된다.
4. 삭제하려는 센터에 연결된 창고가 있으면 창고를 먼저 삭제하라는 안내가 표시되고 삭제가 거부된다.
5. 위 조건을 모두 통과하면 센터가 비활성화 처리되고, 해당 센터의 사용자 권한 데이터가 삭제된다.

### 시나리오 E: 대행의뢰센터 정보 수정 (팝업 — 위탁 기능)

1. 자사 물류대행 여부 설정 팝업(MDBZ01P01)을 열면 자사의 센터 목록이 표시된다.
2. 물류대행 여부를 사용으로 변경하려면 주소, 주소상세, 전화번호가 모두 입력되어야 한다.
3. 저장하면 센터 정보가 업데이트되고 팝업이 닫힌다.

### 시나리오 F: 물류대행업체 검색 및 위탁 의뢰 (팝업 — 위탁 기능)

1. 물류대행업체검색 팝업(MDBZ01P02)에서 사업장명, 물류센터명, 주소로 대행 가능한 업체를 검색한다.
2. 검색 결과에서 원하는 센터를 선택하고 요청 내용을 입력한 뒤 위탁 요청 버튼을 클릭한다.
3. 이미 신청 중이거나 수락된 센터에 재신청하면 이미 처리된 건임을 알리는 오류가 발생한다.
4. 거절된 이력이 있는 경우에는 재신청이 가능하다 (기존 이력이 새 신청으로 덮어써짐).

### 시나리오 G: 위탁 의뢰 수락/거절 (대행업체 측)

1. 대행업체 측 사용자는 대행의뢰신청업체 조회(MDBZ01P01 연동)를 통해 의뢰 목록을 확인한다.
2. 수락 또는 거절 처리를 수행한다.
3. 수락 시 해당 의뢰 상태가 승인 완료로 변경되고, 사업장-사업장 관계와 사업장-창고 연결이 자동으로 생성된다.

## 6. 상태 흐름

### 위탁 의뢰 상태 전이

```
[신청 가능]
     │ 위탁 의뢰 신청
     ▼
[신청중 (REQUEST)]
     │
     ├── 수락 ──────►  [승인 완료 (ACCEPT)]
     │                       │
     │                       └── (의뢰자가 취소 불가: 이미 수락됨)
     │
     ├── 거절 ──────►  [거절 (DENIED)]
     │                       │
     │                       └── 재신청 가능 → 다시 [신청중]으로
     │
     └── 취소 ──────►  [삭제됨]
          (의뢰자 본인만 가능, REQUEST 상태일 때만)
```

| 상태값 | 의미 | 비고 |
|---|---|---|
| REQUEST | 신청 중 | 의뢰 보낸 후 대행업체 응답 대기 중 |
| ACCEPT | 승인 완료 | 대행업체가 수락함 |
| DENIED | 거절됨 | 대행업체가 거절함, 재신청 가능 |
| 신청 가능 | 아직 의뢰하지 않음 | 데이터가 없는 상태 |

## 7. 핵심 업무 규칙

**BR-1. 사업장명 필수**
사업장 기본정보 저장 시 사업장명은 반드시 입력되어야 한다. 비어 있으면 저장이 거부된다.

**BR-2. 센터명 필수 및 사업장 내 고유**
센터를 추가하거나 수정(사용 중 상태인 경우)할 때 센터명은 필수이며, 동일 사업장 내에서 중복될 수 없다.

**BR-3. 최소 하나의 사용 센터 유지**
센터 저장 처리 후 해당 사업장에 사용 중인 센터가 하나도 없으면 저장이 거부된다. 사업장은 반드시 하나 이상의 활성 센터를 보유해야 한다.

**BR-4. 위탁 계약 존재 시 센터 비활성화 제한**
사용 중지 처리하거나 물류대행 여부를 해제하려는 센터에 활성화된 위탁 계약(다른 사업장과의 센터 공유 관계)이 있으면 해당 변경이 거부된다.

**BR-5. 창고 보유 센터 삭제 제한**
삭제하려는 센터에 창고가 연결되어 있으면 창고를 먼저 삭제해야 한다. 창고가 있는 상태로 센터를 삭제할 수 없다.

**BR-6. 권한 보유 사용자 존재 시 센터 삭제 제한**
삭제하려는 센터에 슈퍼 권한이 아닌 일반 사용자의 접근 권한이 있으면 센터 삭제가 거부된다.

**BR-7. 물류대행 여부 사용 시 주소/전화번호 필수**
센터를 물류대행 가능 상태로 설정(등록 또는 수정 시)할 때는 주소, 주소상세, 전화번호가 모두 입력되어야 한다.

**BR-8. 위탁 의뢰 취소는 신청 중 상태에서만 가능**
의뢰자가 의뢰를 취소하려면 해당 의뢰가 신청 중(REQUEST) 상태여야 한다. 이미 수락되거나 거절된 경우 취소할 수 없다.

**BR-9. 센터 신규 등록 시 기본 창고 자동 생성**
새 센터를 등록하면 시스템이 자동으로 기본 창고 템플릿을 복제하여 해당 센터에 기본 창고와 기본 위치를 생성한다.

**BR-10. 신규 센터에 슈퍼 권한 사용자 자동 접근 허용**
새 센터를 등록하면 현재 사업장의 슈퍼 권한 사용자 전원에게 해당 센터 접근 권한이 자동으로 부여된다.

**BR-11. 권한별 사업장 접근 범위**
- 슈퍼 권한: 조건 없이 수정 가능 사업장 목록 전체 접근
- 사업장 권한: 본인이 소속된 사업장만 접근
- 센터 권한: 본인이 소속된 센터가 속한 사업장만 접근
- 화주(배송 의뢰인) 구분의 사업장은 목록에서 제외됨

## 8. 권한별 업무 범위

| 권한 | 사업장 조회 범위 | 기본정보 수정 | 센터 관리 |
|---|---|---|---|
| 슈퍼 | 조건 없이 전체(화주 사업장 제외) | 가능 | 가능 |
