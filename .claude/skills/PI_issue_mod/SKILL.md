# 레드마인 이슈 작업이력 수정 [PI_issue_mod]

인수: **$ARGUMENTS**

---

## 파라미터 파싱

`$ARGUMENTS`는 아래 형식으로 전달된다.

```
{이슈번호} {진행율} {작업내역}
```

예시:
```
/PI_issue_mod 123 80 오늘 API 개발 완료, 내일 테스트 예정
```

파싱 규칙:
- **이슈번호**: 첫 번째 토큰 (숫자, `#` 접두사가 있으면 제거)
- **진행율**: 두 번째 토큰 (0~100 숫자)
- **작업내역**: 세 번째 토큰 이후 나머지 전체 (공백 포함)

---

## 실행 절차

### 1단계 — 파라미터 확인

`$ARGUMENTS`에서 위 규칙으로 세 값을 추출한다.

파라미터가 부족하면 아래 형식으로 안내하고 중단한다.

```
사용법: /PI_issue_mod {이슈번호} {진행율} {작업내역}

예시: /PI_issue_mod 123 80 오늘 API 개발 완료
```

이슈번호·진행율·작업내역 중 누락된 값이 있으면 `AskUserQuestion` 으로 부족한 값을 한 번에 묻는다.

### 2단계 — 레드마인 이슈 수정

`mcp__redmine__updateIssue` 도구를 호출한다.

| 파라미터 | 값 |
|---|---|
| `issue_id` | 파싱한 이슈번호 (정수) |
| `done_ratio` | 파싱한 진행율 (정수, 0~100) |
| `notes` | 파싱한 작업내역 (문자열) |

### 3단계 — 결과 보고

성공 시:
```
✓ 레드마인 이슈 수정 완료

이슈번호 : #{이슈번호}
진행율   : {진행율}%
작업내역 : {작업내역}
```

실패 시 오류 내용을 출력하고 재시도 방법을 안내한다.
