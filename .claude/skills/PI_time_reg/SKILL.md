# 레드마인 이슈 작업시간 등록 [PI_time_reg]

인수: **$ARGUMENTS**

---

## 파라미터 파싱

`$ARGUMENTS`는 아래 형식으로 전달된다.

```
{이슈번호} {시간} {작업내역}
```

예시:
```
/PI_time_reg #16129 4 입고예정 화면 개발
```

파싱 규칙:
- **이슈번호**: 첫 번째 토큰, `#` 접두사 제거 후 정수 변환 (예: `#16129` → `16129`)
- **시간**: 두 번째 토큰, 소수점 허용 (예: `4`, `1.5`, `0.5`)
- **작업내역**: 세 번째 토큰 이후 나머지 전체 (공백 포함)

---

## 실행 절차

### 1단계 — 파라미터 확인

`$ARGUMENTS`에서 위 규칙으로 세 값을 추출한다.

파라미터가 부족하면 아래 형식으로 안내하고 중단한다.

```
사용법: /PI_time_reg {이슈번호} {시간} {작업내역}

예시: /PI_time_reg #16129 4 입고예정 화면 개발
      /PI_time_reg 16129 1.5 단위테스트 작성
```

이슈번호·시간·작업내역 중 누락된 값이 있으면 `AskUserQuestion` 으로 부족한 값을 한 번에 묻는다.

### 2단계 — 레드마인 작업시간 등록

`mcp__redmine__createTimeEntry` 도구를 호출한다.

| 파라미터 | 값 |
|---|---|
| `issue_id` | 파싱한 이슈번호 (정수) |
| `hours` | 파싱한 시간 (소수점 허용) |
| `comments` | 파싱한 작업내역 (문자열) |
| `spent_on` | 오늘 날짜 (YYYY-MM-DD) |

### 3단계 — 결과 보고

성공 시:
```
✓ 레드마인 작업시간 등록 완료

이슈번호 : #{이슈번호}
작업시간 : {시간}h
작업일   : {오늘날짜}
작업내역 : {작업내역}
```

실패 시 오류 내용을 출력하고 재시도 방법을 안내한다.
