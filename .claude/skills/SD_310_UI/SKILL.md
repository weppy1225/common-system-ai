# 화면요건 문서 대화형 작성 [SD_310_UI]

메뉴코드: **$ARGUMENTS**

`deliverables/20-sources/UI/ui-template.md` 규칙에 따라 사용자와 인터뷰하며 `30-domain/30-wms-business/{메뉴코드}/{메뉴코드}-02-ui.md` 를 생성한다.

---

## 시작 전 준비
1. `deliverables/20-sources/UI/ui-template.md` 읽기 (출력형식·저장경로·컴포넌트 기준·정렬 기준 등등)
2. `$ARGUMENTS` 가 비어있으면 첫 질문에서 메뉴코드 수집
3. 진행 중 사용자가 "패스" 또는 "없음" 입력 시 해당 항목 생략

---

## 인터뷰 진행 규칙

- **단계별로 진행** — 한 번에 모든 것을 묻지 않는다
- **각 단계 시작 시** `[N/6단계] 제목` 헤더 표시
- **사용자 답변 후** 즉시 다음 단계로 넘어간다 (중간 요약 없이)
- **6단계 완료 후** 전체 요약을 보여주고 확인 받은 뒤 파일 생성

---

## [1/6단계] 기본정보

아래 4가지를 한 번에 묻는다.

```
[1/6단계] 기본정보

다음 정보를 알려주세요.

1. 메뉴코드 (예: MDPR01)
2. 메뉴명 (예: 프로모션)
3. 메뉴그룹명 / 그룹코드 (예: 기준정보 / MD8000)
4. 화면 목적 (한 줄 설명)
```

---

## [2/6단계] 화면 구조

```
[2/6단계] 화면 구조

UI 유형을 선택하고 각 그리드의 이름을 알려주세요.

A. 단일 그리드
   → 예: 목록 1개, 조회 후 처리 버튼

B. 상단+하단 2그리드 (헤더/디테일)
   → 예: 상단: 주문 목록 선택 시 / 하단: 주문상품 로드

C. 상단+하단 좌우 분할 3그리드
   → 예: 상단: 프로모션 / 하단좌: 대상상품 / 하단우: 증정품목

D. 기타 (직접 설명)

선택 + 그리드명 함께 입력하세요.
```

---

## [3/6단계] 검색 영역

```
[3/6단계] 검색 영역

검색 조건 항목을 나열해주세요. (없으면 "없음")

1행 5컬럼 배치 기준. 사업장은 자동으로 1행 1번 고정.

컴포넌트 힌트:
- (공통코드) → Select
- (숫자) → Input - 숫자
- (날짜) → Date
- (품목팝업) → Input + 팝업 🔍
- 표기 없으면 → Input

비고(disable 조건 등)는 항목 뒤에 괄호로.
예: 대분류(공통코드, 대상구분=품목일때 비활성)

예시:
1행: 사업장 / 프로모션명 / 프로모션유형(공통코드) / 증정유형(공통코드) / 대상구분(공통코드)
2행: 대상수량(숫자) / 기준일자(날짜) / 시작일자(날짜) / 종료일자(날짜)
3행: 품목번호(품목팝업) / 품목명(품목팝업) / 대분류(공통코드, 대상구분=품목일때 비활성)
```

## [4/6단계] 그리드 상세

각 그리드를 **순서대로** (상단 → D1 → D2) 아래 형식으로 묻는다.

```
[4/6단계] {그리드명} 상세

■ 툴바 버튼
조작 패턴을 선택하세요 (복수 선택 가능):
- 추가(팝업) / 수정(팝업) / 삭제
- 행추가 / 행삭제 / 저장 (인라인 편집 그리드)
- 특수 버튼 (이름과 동작 설명)

비활성(disabled) 조건 있으면 함께 알려주세요.

■ 그리드 컬럼
컬럼 목록을 나열해주세요.
형식: 컬럼명 (정렬: 좌/중/우) [비고]

정렬 생략 시 자동 판단:
- 이름·명칭·설명 → 좌측
- 코드·번호·날짜·상태 → 중앙
- 수량·금액·수치 → 우측

비고 예시: 숫자, 날짜, 체크아이콘, 조건부편집(인제), 팝업연동, 자동채움
PK·순번 컬럼 제외해도 됩니다.
```

하단 분할 그리드(D1, D2)가 있으면 동일한 질문을 반복한다.

## [5/6단계] 등록/수정 팝업

```
[5/6단계] 등록/수정 팝업

팝업(Edt.vue)이 있나요? (없으면 "없음")

있다면 아래 정보를 알려주세요.

■ 팝업 필드 목록
형식: 항목명 (컴포넌트) [필수여부] [disabled 조건]

예:
- 사업장 (Select) 필수, 수정시 비활성
- 프로모션명 (Input) 필수
- 프로모션유형 (Select) 필수
- 증정유형 (Select) 필수, 프로모션유형=금액이면 한 건만 고정
- 비고 (Textarea) 전체너비

■ 팝업 안내문구 (안내 문구, 없으면 생략)
```

## [6/6단계] 업무규칙

```
[6/6단계] 업무규칙

특별한 업무규칙이 있으면 알려주세요. (없으면 "없음")

구분:
- 순서강제: A 완료 후 B 등록 가능
- 수량제한: 특정 조건에서 N개 초과 불가
- 값강제: X값이면 Y값 자동 고정
- UI disable: 조건에 따라 필드/컬럼 비활성
- 특수기능: 자동계산, 복사, 배치생성 등

예:
1. 프로모션 먼저 등록 후 D1·D2 등록 (순서강제)
2. 증정유형=여러번이면 D1 1개만 등록 가능 (수량제한)
3. 프로모션유형=금액이면 증정유형은 한번만 선택 (값강제)
```

## 전체 요약 확인

6단계 완료 후 아래 형식으로 요약 출력 후 확인 받는다.

```
=== 화면요건 요약 ===

메뉴: {코드} {메뉴명} ({그룹명} / {그룹코드})
목적: {목적}
UI유형: {유형}

검색조건: {N}개 항목 ({N}행)
그리드: {그리드명 목록}
팝업: {있음(필드 N개) / 없음}
업무규칙: {N}개
저장 경로: 30-domain/30-wms-business/{메뉴코드}/{메뉴코드}-02-ui.md

이대로 {메뉴코드}-02-ui.md를 생성할까요?
(수정할 부분이 있으면 항목명과 내용을 알려주세요)
```

확인 후 생성. 수정 요청 시 해당 부분만 재질문.

## ui.md 생성 규칙

`deliverables/20-sources/UI/ui-template.md` 의 출력 형식·컴포넌트 표기·정렬 기준·업무규칙 구분값을 따른다.

### 컴포넌트 표기 변환
| 사용자 입력                  | ui.md 표기      |
| ---------------------------- | --------------- |
| 공통코드 / Select / 드롭다운 | Select          |
| Input / 텍스트               | Input           |
| 숫자 / number                | Input - 숫자    |
| 날짜 / Date / 달력           | Date            |
| 품목팝업 / 팝업🔍            | Input + 팝업 🔍 |
| Textarea                     | Textarea        |

### 그리드 컬럼 고정 규칙

모든 그리드 컬럼 처음 2개:

- 순번 1: No. (중앙)
- 순번 2: checkbox (중앙)

### 검색영역 빈칸 처리

5컬럼 기준, 행이 채워지지 않은 나머지 칸은 `(빈칸)` 으로 명시.

### 저장
```
30-domain/30-wms-business/{메뉴코드}/{메뉴코드}-02-ui.md
```

디렉토리 없으면 생성 후 저장. 완료 후 경로 출력.
