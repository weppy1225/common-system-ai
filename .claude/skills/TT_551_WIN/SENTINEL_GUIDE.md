# DB 이관계획서 템플릿 Sentinel 셋업 가이드

`TT_551_WIN` 스킬이 동작하려면 **`template/05 이행(TT)/DB이관계획서_템플릿.xlsx`** 에 sentinel placeholder 가 박혀 있어야 합니다. 이 가이드는 **최초 1회만** 따라하면 됩니다 — 한 번 셋업하면 이후 매 고객사마다 자동 채워집니다.

---

## 1단계 — 템플릿 파일 배치

```
원본:  input/TT.551/TT_551_DB이관계획서_20260511.xlsx
대상:  template/05 이행(TT)/DB이관계획서_템플릿.xlsx
```

복사 (PowerShell):
```powershell
Copy-Item "input\TT.551\TT_551_DB이관계획서_20260511.xlsx" `
          "template\05 이행(TT)\DB이관계획서_템플릿.xlsx"
```

> 원본은 그대로 두고 복사본을 템플릿으로 사용합니다.

---

## 2단계 — 스칼라 Sentinel 박기

엑셀을 열고 자동으로 채워질 셀의 **기존 값을 지우고** 아래 sentinel 텍스트로 교체합니다.

### 표지·헤더 영역 (개별 셀)

| Sentinel | 의미 | 예시 결과 |
|---|---|---|
| `{{customer}}` | 고객사명 | `가나다물류` |
| `{{author}}` | 담당자 | `홍길동` |
| `{{generated_at}}` | 산출물 생성일시 | `2026-05-12 14:33` |
| `{{plan_date}}` | 이관 예정일자 | `2026-05-20` |

### 원본 DB 정보 영역

| Sentinel | 의미 | 예시 결과 |
|---|---|---|
| `{{source_host}}` | DB host | `192.168.10.20` |
| `{{source_db}}` | DB 이름 | `wms-cloud-test` |
| `{{source_schema}}` | schema | `public` |
| `{{source_pg_version}}` | PG 버전 | `15.4` |

### 요약 영역

| Sentinel | 의미 | 예시 결과 |
|---|---|---|
| `{{group_count}}` | 그룹 개수 | `3` |
| `{{table_count}}` | 테이블 개수 | `5` |
| `{{row_count_total}}` | 전체 row 합계 | `244` |

### 메모 영역 (텍스트 박스 또는 큰 셀)

| Sentinel | 의미 | 사람 입력 |
|---|---|---|
| `{{precondition}}` | 사전 조건 | 여러 줄 가능 (DB 백업·점검 모드 등) |
| `{{rollback}}` | 롤백 계획 | 여러 줄 가능 |
| `{{warnings}}` | 마커 누락 경고 (자동) | `sm_legacy_temp, mdm_obsolete` |

> 사람이 입력할 항목(`precondition`, `rollback`) 도 sentinel 로 박아두면 `TT_551_WIN` 이 실행 시 `AskUserQuestion` 으로 받은 텍스트를 자동 치환합니다.

---

## 3단계 — 표 반복 행 마커 박기

표 데이터가 들어갈 위치는 **첫 데이터 행의 첫 셀**에 마커를 박습니다. 그 행이 첫 행이 되고, 데이터 개수만큼 아래로 자동 행 삽입.

### 그룹 요약 표 (행 단위 = 그룹별 1행)

표의 **첫 데이터 행의 첫 컬럼 셀**에 다음을 입력:

```
{{rows:groups}}
```

같은 행에서 우측으로 컬럼이 순서대로 채워집니다 (좌→우):

| 컬럼 위치 | 들어가는 값 | 예시 |
|---|---|---|
| 1번째 (마커 셀) | `no` (순번) | `1` |
| 2번째 | `group_key` | `01_common_code` |
| 3번째 | `group_desc` | `공통코드` |
| 4번째 | `table_count` | `2` |
| 5번째 | `row_sum` | `99` |
| 6번째 | `sql_file` | `01_common_code.sql` |
| 7번째 | `file_size_kb` | `8.2` |

### 테이블 상세 표 (행 단위 = 테이블별 1행)

```
{{rows:tables}}
```

| 컬럼 위치 | 들어가는 값 | 예시 |
|---|---|---|
| 1번째 (마커 셀) | `no` | `1` |
| 2번째 | `group_desc` | `공통코드` |
| 3번째 | `table_name` | `sm_comm_h` |
| 4번째 | `table_desc` | `공통코드 헤더` |
| 5번째 | `rows` | `12` |
| 6번째 | `fk_parent` | `(빈칸 또는 sm_comm_h)` |
| 7번째 | `sql_file` | `01_common_code.sql` |
| 8번째 | `apply_order` | `1` |

## 4단계 — 표 컬럼 수가 다를 때

표의 컬럼 수가 위 dict 의 키 수와 다를 수 있습니다.

- **표 컬럼이 더 적으면** → 좌측부터 매칭, 남는 키는 무시됨 (콘솔에 경고)
- **표 컬럼이 더 많으면** → 좌측부터 매칭, 남는 셀은 그대로 (또는 사람이 직접 채울 영역)

원하지 않는 컬럼은 빼고 싶다면, 마커 위치를 옮기거나 데이터 dict 자체를 수정해야 합니다 (스킬 코드 수정 필요). 단순 셋업이라면 **표의 컬럼 순서를 위 표에 맞게 조정**하는 게 가장 빠릅니다.

---

## 5단계 — 머지셀 주의

- 마커 셀이 머지셀의 **좌상단** 에 있어야 안전합니다.
- 표 안에 머지된 헤더가 있어도 OK (헤더는 영향 없음).
- 데이터 행 안에 머지셀이 있으면 자동 행 삽입 시 머지 패턴도 복제됩니다.

복잡한 머지 구조는 가능한 피하고, **표 본문은 단순 격자**로 두는 걸 추천합니다.

---

## 6단계 — 셋업 검증

엑셀에 sentinel 을 모두 박은 후, **저장하고 닫은 다음** `/TT_551_WIN` 을 실행합니다. 실행 결과 콘솔에서:

```
[TT_551_WIN] {시트명}!{셀좌표}: rows:groups → 3행 채움
[TT_551_WIN] {시트명}!{셀좌표}: rows:tables → 5행 채움
[TT_551_WIN] 엑셀 채우기 완료: output\05 이행(TT)\TT_551_DB이관계획서_xxx.xlsx
  스칼라 치환:  13건
  표 행 채움:   8건  (2 마커)
```

생성된 엑셀을 열어 자동 채워진 내용 확인 후 미세 조정. 위치가 마음에 안 들면 템플릿에서 sentinel 셀만 이동하면 됩니다.

## 트러블슈팅

| 문제 | 원인 | 해결 |
|---|---|---|
| 스칼라가 치환 안 됨 | `{{customer}}` 가 텍스트박스 안에 있음 | 텍스트박스 sentinel 치환은 미지원 — 셀로 옮기기 |
| 표 행이 안 늘어남 | 마커 셀이 머지셀 우측·하단에 있음 | 머지셀 좌상단으로 이동 |
| 헤더 텍스트도 같이 치환됨 | 헤더 셀에 sentinel 비슷한 텍스트가 있음 | 헤더 텍스트를 sentinel 형식과 겹치지 않게 |
| 빈 행이 남음 | data_rows 가 0개인 경우 마커는 비워짐 | 정상 동작 (해당 그룹/테이블이 없을 때) |
| sentinel 이 표시됨 | scalars dict 에 해당 key 가 없음 | `fill_data.json` 확인 / 오타 확인 |

## (옵션) Sentinel 자동 삽입 스크립트

원본 엑셀에 sentinel 을 자동으로 박아주는 스크립트는 **현재 미제공** 입니다. 셀 좌표가 템플릿마다 다르기 때문입니다. 한 번 셋업해두면 영구적으로 재사용되므로, 첫 사용 시 수동으로 박는 것이 가장 안전합니다.

향후 표준 템플릿이 확정되면 `scripts/setup_template.py` 형태로 자동화 가능합니다.
