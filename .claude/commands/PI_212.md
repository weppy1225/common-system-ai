# 단위테스트 보고서 생성 명령어

업체명: **$ARGUMENTS**

BE / FE 소스를 스캔하여 `scripts/gen_wms_unit_test_report.js`를 최신 상태로 재작성한 뒤 Excel 보고서를 생성한다.

---

## 입출력 경로

| 구분 | 경로 |
|---|---|
| BE 소스 | `C:\zinide\workspace_cloud\cloud-wms-be` |
| FE 소스 | `C:\zinide\workspace_cloud\cloud-wms-fe` |
| 생성 스크립트 | `scripts/gen_wms_unit_test_report.js` |
| 템플릿 | `template\04 구현(PI)\PI.212-단위테스트보고서.xlsx` |
| 출력 | `output\04 구현(PI)\PI.212-단위테스트보고서_{업체명}_{YYMMDD}.xlsx` |

- `{업체명}`: $ARGUMENTS (없으면 `WMS`)
- `{YYMMDD}`: 오늘 날짜

---

## 스크립트 재작성 규칙

**매번 실행 시 아래 단계를 수행하여 스크립트를 완전히 새로 작성한다.**
소스에 변경이 없어도 동일하게 수행한다.

### BE 테스트 수집

`C:\zinide\workspace_cloud\cloud-wms-be` 하위 `ZTEST_*.java` 파일을 전부 읽는다.

- `@Test` 어노테이션이 붙은 메서드만 수집한다. (주석 처리된 것 제외)
- 각 테스트 항목의 속성:
  - **플랫폼**: `WEB` (인터페이스 관련 패키지는 `I/F`)
  - **대메뉴**: 패키지 경로(lv2)로 결정
    - `md8000` → `기준정보`
    - `iw1000` → `입고`
    - `rt2000` → `반품`
    - `iv3000` / `iv3100` / `iv3200` → `재고`
    - `ow5000` → `출고`
    - `mm9200` / `sm9000` → `시스템관리`
    - `if9100` → `인터페이스` (플랫폼도 `I/F`)
  - **테스트ID**: `WMS-BE-001` 형식으로 순번 자동 채번
  - **메뉴**: `클래스명에서 코드 추출(메뉴코드)` 형식 (예: `사업장관리(MDBZ01)`)
  - **구분**: `기능`
  - **내용**: `@DisplayName` 값 → 없으면 메서드명을 한글로 풀어서 작성
  - **결과**: `O`

### FE 화면 수집

`C:\zinide\workspace_cloud\cloud-wms-fe\src\views` 하위 `.vue` 파일을 전부 읽는다.

- 각 `.vue` 파일 = 화면 1개로 간주
- 화면 1개당 2~3개의 테스트 항목 생성:
  1. `화면 로딩 - {메뉴명} 조회 결과 정상 표시`
  2. `{메뉴명} 검색 - 조건별 검색 결과 확인` (조회 화면)
  3. `{메뉴명} 등록/수정/삭제 - 기능 정상 동작 확인` (CRUD 화면)
- 각 항목의 속성:
  - **플랫폼**: `WEB` (스캔 화면 `*sc01.vue`는 `PDA`, 인터페이스 `if9100` 경로는 `I/F`)
  - **대메뉴**: 폴더 구조(lv2)로 결정 (BE와 동일 기준)
  - **테스트ID**: `WMS-FE-001` 형식으로 순번 자동 채번
  - **메뉴**: `{화면명}({파일명})` 형식 (예: `사업장관리(mdbz01)`)
  - **구분**: `기능`
  - **결과**: `O`

## 스크립트 고정 상수 업데이트

스크립트 상단의 아래 두 상수를 오늘 날짜와 업체명에 맞게 수정한다.

```javascript
const DATE_SERIAL = {오늘날짜의 Excel 시리얼 숫자};  // YYYY-MM-DD
const REPORTER = '신현규';
```

출력 파일명 상수도 업데이트:
```javascript
const OUTPUT = path.join(__dirname, '../output/04 구현(PI)/PI.212-단위테스트보고서_{업체명}_{YYMMDD}.xlsx');
```

## 실행

스크립트 재작성 완료 후 실행:

```powershell
node "C:\zinide\workspace_cloud\cloud-wms-doc\scripts\gen_wms_unit_test_report.js"
```

---

## 주의사항

- 템플릿 양식(라인 사이즈, 색상, 병합셀, 인쇄설정)을 절대 변경하지 않는다.
  `xlsx-populate`로 데이터만 교체한다.
- `xlsx-populate` 패키지가 없으면 먼저 설치: `npm install xlsx-populate`
- 스크립트의 `TESTS` 배열을 **전체 교체**한다. 기존 항목을 일부만 수정하지 않는다.
- BE `@Test` 중 주석처리(`//`)된 메서드는 수집하지 않는다.
