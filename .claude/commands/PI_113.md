# 공통코드정의서 생성 명령어

업체명: **$ARGUMENTS**

BE 프로젝트의 공통코드 MD 파일을 읽어  
`template/04 구현(PI)/PI.113-공통코드정의서.xlsx` 양식으로 산출물을 생성한다.

---

## 입출력 경로

| 구분 | 경로 |
|---|---|
| MD 소스 | `C:\zinide\workspace_cloud\cloud-wms-be\DEV_DOC\ai-docs\10-database\90-schema\30-data\01-common-code.md` |
| 템플릿 | `template\04 구현(PI)\PI.113-공통코드정의서.xlsx` |
| 출력 | `output\04 구현(PI)\PI.113-공통코드정의서_{업체명}_{YYMMDD}.xlsx` |

- `{업체명}`: $ARGUMENTS
- `{YYMMDD}`: 오늘 날짜 (예: 260506)
- 출력 폴더가 없으면 생성

---

## MD 파일 구조 (파싱 기준)

```
## 1. 코드 헤더 목록
| 코드(comm_h_cd) | 코드명(comm_h_nm) | 사용자코드(user_cd_yn) | 수정가능여부(user_edit_yn) | 사용여부(use_yn) | 수불코드(inout_cd) |
| [CODE_NAME](#anchor) | 코드명 | N | N | Y | |   ← 링크 형식 주의

## 2. 코드 상세 목록

### HEADER_CD
**코드명**
| 코드 | 코드명 | 참조헤더 | 참조코드 | 표시순서 | 사용여부 | 비고 |
| 11 | 예정 | | | 1 | Y | |
```

> 파싱 주의:
> - 헤더 테이블의 코드값은 `[CODE](#anchor)` 링크 형식 → `CODE`로 변환
> - 구분선 행(`|---|---|` 패턴) 스킵
> - 일부 코드 상세 테이블은 "기본값" 컬럼이 추가되어 8컬럼인 경우 있음
> - MD 파일이 크므로 (`Get-Content -Encoding UTF8`) 인코딩 필수

## Excel 출력 구조 (6개 시트)

### 시트 1: 표지
- 템플릿 내용 유지 (수정 불필요)

### 시트 2: 개정이력
- 템플릿 내용 유지 (수정 불필요)

### 시트 3: 3.코드그룹 — SM_COMM_H 데이터

| 열 | 컬럼명 | DB 컬럼명 | 입력값 |
|---|---|---|---|
| A | 사업장 | BIZ_SEQ | `1` (고정) |
| B | 그룹코드 | COMM_H_CD | `comm_h_cd` |
| C | 코드그룹명 | COMM_H_NM | `comm_h_nm` |
| D~F | 코드설명 (3열 병합) | — | `comm_h_nm` 값으로 대체 |
| G | 사용자코드여부 | USER_CD_YN | `user_cd_yn` |
| H | 수불유형여부 | INOUT_CD | `inout_cd` |
| I | 비고 | — | 빈값 |

- **R1~R6**: 문서 메타 헤더 (건드리지 않음)
- **R7**: 컬럼 헤더 행 (배경색 `Interior.Color=15849925`, Bold) — 수정 불필요
- **R8~**: 기존 데이터 삭제 후 신규 입력
- D~F 열 병합: 각 데이터 행마다 `Range(Cells(row,4), Cells(row,6)).MergeCells = $true`

### 시트 4: 4.상세코드 — SM_COMM_D 데이터

| 열 | 한글명(R2) | DB컬럼명(R3) | 입력값 |
|---|---|---|---|
| A | 사업장 | BIZ_SEQ | `1` (고정) |
| B | 그룹코드 | COMM_H_CD | `comm_h_cd` (현재 헤더) |
| C | 코드 | COMM_D_CD | `코드` |
| D | 코드명 | COMM_D_NM | `코드명` |
| E | 참조상위코드 | REF_H_CD | `참조헤더` |
| F | 참조코드 | REF_D_CD | `참조코드` |
| G | 순서 | DISP_IDX | `표시순서` (숫자) |
| H | 표시여부 | DISP_YN | `Y` (고정) |
| I | 사용여부 | USE_YN | `사용여부` |

- **R1**: `■ 상세코드` (섹션 제목, 수정 불필요)
- **R2**: 한글 컬럼 헤더 (배경색 `Interior.Color=15849925`, Bold) — 수정 불필요
- **R3**: DB 컬럼명 헤더 (배경색 동일) — 수정 불필요
- **R4~**: 기존 데이터 삭제 후 신규 입력 (헤더별 상세 코드를 순서대로)

### 시트 5: 그룹SQL — SM_COMM_H INSERT SQL

A열에 SQL 문을 1행씩 입력한다.

```sql
INSERT INTO sm_comm_h (biz_seq, comm_h_cd, comm_h_nm, user_cd_yn, user_edit_yn, inout_cd, use_yn, reg_id, reg_dt)
VALUES (1, '{comm_h_cd}', '{comm_h_nm}', '{user_cd_yn}', '{user_edit_yn}', {inout_cd_or_NULL}, '{use_yn}', 'system', now());
```

- `inout_cd`가 빈값이면 → `NULL` (따옴표 없음)
- 코드명에 작은따옴표(`'`) 포함 시 → `''`로 이스케이프

### 시트 6: 상세SQL — SM_COMM_D INSERT SQL

A열에 SQL 문을 1행씩 입력한다.

```sql
INSERT INTO sm_comm_d (biz_seq, comm_h_cd, comm_d_cd, comm_d_nm, ref_h_cd, ref_d_cd, disp_no, disp_yn, use_yn, reg_id, reg_dt)
VALUES (1, '{comm_h_cd}', '{comm_d_cd}', '{comm_d_nm}', {ref_h_or_NULL}, {ref_d_or_NULL}, {disp_no}, 'Y', '{use_yn}', 'system', now());
```

- `ref_h_cd`, `ref_d_cd`가 빈값이면 → `NULL`
- `disp_no`는 숫자 (따옴표 없음), 빈값이면 → `0`
- 코드명에 작은따옴표(`'`) 포함 시 → `''`로 이스케이프

## 실행 절차

### 1단계 — MD 파싱

```powershell
$mdPath = "C:\zinide\workspace_cloud\cloud-wms-be\DEV_DOC\ai-docs\10-database\90-schema\30-data\01-common-code.md"
$content = Get-Content $mdPath -Encoding UTF8

$headers = @()
$details = [ordered]@{}
$section = ""; $currentHcd = ""

foreach ($line in $content) {
    if ($line -match "^##\s+1\.") { $section = "header"; continue }
    if ($line -match "^##\s+2\.") { $section = "detail"; continue }
    if ($line -match "^##\s+[3-9]") { $section = "other"; continue }

    if ($section -eq "detail" -and $line -match "^###\s+(\S+)") {
        $currentHcd = $Matches[1]
        if (-not $details.Contains($currentHcd)) { $details[$currentHcd] = @() }
        continue
    }

    if ($line -match "^\|") {
        if ($line -match "^\|[-:\s|]+\|$") { continue }
        $cells = $line -split '\|' | ForEach-Object { $_.Trim() } | Where-Object { $_ -ne "" }
        if ($cells.Count -lt 2) { continue }

        if ($section -eq "header") {
            if ($cells[0] -match "코드|comm_h") { continue }
            $cd = $cells[0] -replace '\[([^\]]+)\]\([^)]+\)', '$1'
            $headers += [pscustomobject]@{
                cd           = $cd.Trim()
                nm           = if ($cells.Count -gt 1) { $cells[1].Trim() } else { "" }
                user_cd_yn   = if ($cells.Count -gt 2) { $cells[2].Trim() } else { "N" }
                user_edit_yn = if ($cells.Count -gt 3) { $cells[3].Trim() } else { "N" }
                use_yn       = if ($cells.Count -gt 4) { $cells[4].Trim() } else { "Y" }
                inout_cd     = if ($cells.Count -gt 5) { $cells[5].Trim() } else { "" }
            }
        }
        elseif ($section -eq "detail" -and $currentHcd -ne "") {
            if ($cells[0] -eq "코드" -or $cells[0] -eq "코드값") { continue }
            $details[$currentHcd] += [pscustomobject]@{
                h_cd     = $currentHcd
                cd       = $cells[0].Trim()
                nm       = if ($cells.Count -gt 1) { $cells[1].Trim() } else { "" }
                ref_h    = if ($cells.Count -gt 2) { $cells[2].Trim() } else { "" }
                ref_cd   = if ($cells.Count -gt 3) { $cells[3].Trim() } else { "" }
                sort_ord = if ($cells.Count -gt 4) { $cells[4].Trim() } else { "" }
                use_yn   = if ($cells.Count -gt 5) { $cells[5].Trim() } else { "Y" }
            }
        }
    }
}
```

### 2단계 — 파일 준비 및 Excel 열기

```powershell
$projectRoot  = "C:\zinide\workspace_cloud\cloud-wms-doc"
$templatePath = "$projectRoot\template\04 구현(PI)\PI.113-공통코드정의서.xlsx"
$outputDir    = "$projectRoot\output\04 구현(PI)"
$yymmdd       = (Get-Date).ToString("yyMMdd")
$outputPath   = "$outputDir\PI.113-공통코드정의서_$($ARGUMENTS)_${yymmdd}.xlsx"

if (-not (Test-Path $outputDir)) { New-Item -ItemType Directory -Path $outputDir -Force }
Copy-Item -Path $templatePath -Destination $outputPath -Force

$excel = New-Object -ComObject Excel.Application
$excel.Visible = $false; $excel.DisplayAlerts = $false
$wb = $excel.Workbooks.Open($outputPath)
```

### 3단계 — 3.코드그룹 데이터 입력

- `$ws.UsedRange.Rows.Count`로 마지막 행 확인
- R8~ 기존 데이터 삭제: `$ws.Range($ws.Rows.Item(8), $ws.Rows.Item($lastRow)).Delete()`
- R8부터 헤더 데이터 순서대로 입력
- 각 행마다 D:F 3열 병합 후 값 입력

### 4단계 — 4.상세코드 데이터 입력

- R4~ 기존 데이터 삭제
- `$details.Keys` 순서대로 헤더별 상세 코드를 연속 입력

### 5단계 — 그룹SQL / 상세SQL 시트 생성

- 기존 내용 전체 Clear 후 A열에 SQL 1행씩 입력
- 작은따옴표 이스케이프: `$val -replace "'", "''"`
- 빈값 처리: NULL (따옴표 없음)

### 6단계 — 테두리 적용

아래 범위에 얇은 실선 테두리 적용 (`xlContinuous=1, xlThin=2`):
- 3.코드그룹: `A7:I{lastRow}`
- 4.상세코드: `A2:I{lastRow}`

### 7단계 — 인쇄 영역 설정 (전 시트)

여백 공통: 좌우 0.5인치 / 상하 0.7인치 / 헤더·푸터 0.3인치

| 시트 | PrintArea | Orientation | FitToPagesWide | PrintTitleRows |
|---|---|---|---|---|
| 표지 | `$A$1:$M$24` | 1(세로) | 1 | 없음 |
| 개정이력 | `$A$1:$E$21` | 1(세로) | 1 | 없음 |
| 3.코드그룹 | `$A$1:$I${lastRow3}` | 1(세로) | 1 | `$7:$7` |
| 4.상세코드 | `$A$1:$I${lastRow4}` | 1(세로) | 1 | `$2:$3` |
| 그룹SQL | `$A$1:$A${lastRowSql1}` | 2(가로) | 1 | 없음 |
| 상세SQL | `$A$1:$A${lastRowSql2}` | 2(가로) | 1 | 없음 |

모든 시트: `Zoom = $false`, `FitToPagesTall = $false`

### 8단계 — 저장 및 해제

```powershell
$wb.Save()
$wb.Close($false)
$excel.Quit()
[System.Runtime.Interopservices.Marshal]::ReleaseComObject($excel) | Out-Null
[GC]::Collect()
```
