# 요구사항정의서 생성 명령어

업체명: **$ARGUMENTS**

`input/RA.212/` 폴더의 회의록·메뉴현황 Excel 파일을 읽어 요구사항을 분석하고,  
`template/02 분석(RA)/RA.314-요구사항정의서.xlsx` 양식으로 산출물을 생성한다.

---

## 실행 절차

### 1단계 — 입력 파일 확인

`input/RA.212/` 폴더에서 모든 `.xlsx` 파일을 확인한다.

- `RA.212-회의록_*` 파일: 요구사항 도출의 핵심 소스
- `RA.311-메뉴현황_*` 파일: 메뉴 구성 및 I/F 범위 확인

$ARGUMENTS 가 비어 있으면 파일명에서 업체명을 자동 추출한다.  
파일명 패턴: `{문서코드}_{업체명}_{날짜}.xlsx` → 두 번째 `_` 구분자 사이 값.

### 2단계 — Excel 파일 읽기 (PowerShell Excel COM)

```powershell
$excel = New-Object -ComObject Excel.Application
$excel.Visible = $false
$excel.DisplayAlerts = $false
```

읽어야 할 내용:
- **회의록** 각 시트: 회의 차수, 협의 항목, 결정 사항, 요청 사항
- **메뉴현황**: 대분류별 메뉴 목록, I/F 유무, 추가/변경 이력(비고)

### 3단계 — 요구사항 분석 및 도출

읽은 내용을 아래 9개 부문으로 분류하여 요구사항 목록을 작성한다.

| 코드 | 부문 | 도출 기준 |
|---|---|---|
| CO | 공통 | 시스템 구성, 사용자/권한, 알람 등 |
| MD | 기준 | 기준정보 메뉴 및 관련 협의 사항 |
| IW | 입고 | 입고관리 메뉴 및 협의 사항 |
| RM | 반품 | 반품관리 메뉴 및 협의 사항 |
| IV | 재고 | 재고관리·조회·실사 메뉴 및 협의 사항 |
| OW | 출고 | 출고관리 메뉴 및 협의 사항 |
| IF | I/F | ERP·물류사·DAS 등 외부 인터페이스 |
| PDA | PDA | PDA 메뉴 및 협의 사항 |
| ERR | 예외 | 예외 처리, 오류 대응 요건 |

요구사항 ID 체계: `FUR-{코드}-{순번3자리}` (예: FUR-CO-001, FUR-MD-002)

각 요구사항 항목 구성:
- **요구사항명**: 간결한 기능명 (예: "세트해체 처리")
- **내용**: 상세 설명 — 회의 결정사항을 근거로 작성. 임의로 내용을 만들지 않는다.
- **요구처**: 물류팀 / IT솔루션팀 (I/F 항목은 IT솔루션팀)
- **작성일자**: 해당 내용이 협의된 회의 날짜 (YYYY-MM-DD)
- **우선순위**: A(필수) / B(권장) / C(선택)
- **수용여부**: 수용 / 수용불가 / 고려
- **비고**: 특이사항 (미수신 처리, 신규 추가 날짜 등)

> **원칙**: 회의록에 없는 내용을 임의로 만들지 않는다. 메뉴현황에만 있고 회의록에 협의 내용이 없으면 메뉴 존재 사실만 기재한다.

### 4단계 — 출력 파일 생성

#### 4-1. 출력 경로 및 파일명 결정

```
output/02 분석(RA)/RA.314-요구사항정의서_{업체명}_{YYMMDD}.xlsx
```

- `{업체명}`: $ARGUMENTS 또는 파일명에서 자동 추출한 값
- `{YYMMDD}`: 오늘 날짜 (예: 260506)
- 출력 폴더가 없으면 생성

#### 4-2. 템플릿 복사 후 수정

```powershell
$src = "template\02 분석(RA)\RA.314-요구사항정의서.xlsx"
$dst = "output\02 분석(RA)\RA.314-요구사항정의서_{업체명}_{YYMMDD}.xlsx"
Copy-Item -Path $src -Destination $dst -Force
$wb = $excel.Workbooks.Open((Resolve-Path $dst).Path)
```

#### 4-3. 시트별 수정

**표지 시트**
- 셀 중 업체명·날짜 텍스트가 있으면 교체
- 도형/이미지 기반이면 수동 수정 필요 사항으로 보고

**개정이력 시트**
- v1.0 / 최초작성일 / 작성자 유지 또는 갱신

**요구사항 일람표 시트**
- 9개 대분류 코드표 유지 (수정 불필요)

**요구사항정의서 시트**
- 헤더 행(1~3행) 유지
- 4행부터 기존 샘플 데이터 전체 삭제
- 분석한 요구사항을 부문 순서(CO → MD → IW → RM → IV → OW → IF → PDA → ERR)로 4행부터 입력

#### 4-4. 저장 및 닫기

```powershell
$wb.Save()
$wb.Close($false)
$excel.Quit()
[System.Runtime.Interopservices.Marshal]::ReleaseComObject($excel) | Out-Null
```
