# 검색 필터 영역 규칙

화면 상단에 위치하는 검색 조건 입력 영역에 적용되는 규칙이다.

---

## 1. 레이아웃 규칙 (Grid & Layout)

- 한 행(Row)에 최대 **5컬럼** 구조로 배치한다.
- 각 컬럼의 너비는 균등 배분한다.
- 각 행의 높이는 일정하게 고정한다.
- **레이블 + 입력창** 을 하나의 세트로 묶어 반복 배치한다.
- 한행에 5개 컬럼이 채워지지 않으면 **레이블 + 레이블(흰색)**로 채운다.
- 달력의 컴포넌트 시작일자 ~ 종료일자 넣을시 입력칸 사이즈안에 들어갈수 있도록 시작일자 종료일자의 사이즈을 조절한다.
- 검색조건(search-area) 테이블의 각 컬럼 셀(val) 너비를 균등하게 조정한다.
- 검색조건(search-area) 테이블에서 항목이 채워지지 않은 빈 셀은, 레이블(명칭) 위치의 셀은 `#f3f4f6`, 입력 컴포넌트 위치의 셀은 `#ffffff` 로 배경색을 구분하여 채운다. 이를 통해 빈 영역에서도 레이블/입력란의 시각적 구조를 유지한다.
- **검색 영역의 항목은 명세서에 정의된 순번 단위 그대로 1항목 = 1레이블 + 1입력 셀로 배치한다.** 시작일자·종료일자처럼 논리적으로 연관된 항목이라도 명세에서 별도 순번이면 각각 독립된 셀로 배치한다. 임의로 합치거나(`colspan`), 숨기거나(`display:none`) 하지 않는다.

---

## 2. 레이블 규칙 (Label Styling)

- 레이블 배경색은 `#f3f4f6`, 텍스트 색상은 `#374151`을 사용한다. 검색조건이 5컬럼을 채우지 못할 경우, 남은 셀은 레이블(명칭) 영역은 `#f3f4f6`, 입력 영역은 `#ffffff`로 구분하여 채운다.
- 레이블 텍스트는 **우측 정렬**, `white-space: nowrap`으로 줄바꿈을 방지한다.
- 레이블 텍스트는 **굵은 글씨(`font-weight: 600`)**, `font-size: 13px`으로 표시한다.
- 레이블 고정 너비는 `80px`, 우측 패딩은 `12px`으로 설정한다.

---

## 3. 입력 컴포넌트 규칙 (Input Components)

- 입력창(Input)과 선택창(Select Box)은 셀 너비에 맞춰 100% 가변 너비로 설정한다.
- 모든 입력/선택 컨트롤의 높이는 `26px`, `font-size: 13px`, 텍스트 색상 `color: #111827`, `background: #fff`으로 통일한다.
- 셀(td) 높이는 `36px`, 패딩은 `4px 10px`, 테두리 `border: 1px solid #e5e7eb`으로 설정한다.
- 입력/선택 컨트롤의 테두리는 `1px solid #d1d5db`, `border-radius: 4px`, 내부 패딩 `padding: 0 8px`을 적용한다.
- 포커스 시 테두리 색상 `#7aa2c8`, 외곽 glow `box-shadow: 0 0 0 2px rgba(122,162,200,0.15)`을 적용한다.
- 항목 간 구분은 세로선과 가로선으로 명확하게 경계를 형성한다.
- **검색 조건의 Select(셀렉트 박스)에 "전체" 옵션을 임의로 추가하지 않는다.** 명세서에 명시된 옵션만 표시한다.

---

## 4. 헤더 영역 규칙 (Header)

- 화면 좌측 상단에 **메뉴명 [메뉴코드]** 를 표시하고, 바로 옆에 **검색 / 초기화** 버튼을 좌측 정렬로 배치한다. (COMMON_UI 공통 규칙)
- **검색 / 초기화 / 업무규칙** 버튼은 동일한 `btn-search` 스타일을 적용한다: `border: 1px solid #a8bfd8; background: #ffffff; color: #3a6ea5; font-weight: 500; padding: 0; width: 74px; height: 28px; font-size: 13px; border-radius: 5px; justify-content: center`, 호버 시 `border-color: #7aa2c8; background: #f4f8fc; color: #2a5a8f`.
- **업무규칙** 버튼은 헤더 우측에 배치한다(`margin-left: auto`). 검색·초기화와 시각적으로 분리한다.
- 헤더 영역 하단 구분선(border-bottom)은 사용하지 않는다.
- 검색 조건은 논리적 순서로 배치한다.
  - 1행: 사업장, 문서번호 등 식별 정보
  - 2행: 분류, 유형 등 카테고리 정보

---

## 5. 검색 테이블 렌더링 규칙

- 검색 테이블에 `table-layout: fixed`를 적용하여 입력 컬럼의 너비를 균등하게 강제한다.
- 라벨 컬럼은 고정 너비(예: 82px)로 지정하고, 입력 컬럼은 명시적 너비 없이 나머지 공간을 균등 분배한다.

---

## 6. 검색 영역 컨테이너 규칙

- 검색 조건 영역(`.search-area`)은 `background: #fff; border: 1px solid #e5e7eb; border-radius: 6px; overflow: hidden; flex-shrink: 0`으로 감싼다.
- 검색 영역과 헤더 사이 간격은 `margin-top: -2px`로 조정하여 헤더와 자연스럽게 이어지도록 한다.
- `btn-search` 호버 시 `color: #2a5a8f`도 함께 변경한다 (테두리·배경 변경과 동일 조건).
