# 기본설계 (Basic Design) — 사업장·센터·물류대행(3PL) 관리

> **업무 관점**의 설계서입니다. 화면·전산용어·개발기술을 배제하고, **"누가, 무엇을, 왜, 어떤 순서로"** 처리하는지만 기술합니다.
> 본 문서를 상위 입력으로 하여 상세설계서([화면구조](mdbz01-02-screen.md) / [데이터모델](mdbz01-03-data-model.md) / [API](mdbz01-05-api.md) / [BE 흐름](mdbz01-06-be-flow.md) / [FE 흐름](mdbz01-07-fe-flow.md))에서 개발 구현 요소를 다룹니다.
> ⚠️ **개발 착수 전 [확인 필요 사항(Open Issues)](mdbz01-99-issues.md)을 먼저 검토하세요** — 수락/거절·취소·팝업 진입점 등 미연결 기능이 정리되어 있습니다.

## 1. 업무 정의

본 업무는 **회사(사업장)가 자기 물류 거점(센터)을 관리**하고, 나아가 **다른 물류대행 전문회사에게 물류 운영을 맡기는 위탁 거래를 성사**시키는 업무입니다.

하나의 사업장은 다음 두 가지 성격 중 하나를 가집니다.

- **자사물류**: 자기 물류센터를 직접 운영하는 회사
- **물류대행**: 다른 회사의 물류를 대신 처리해 주거나, 자기 물류를 다른 회사에 맡기는 회사

세 가지 업무 영역으로 구성됩니다.

| 업무 영역 | 설명 |
|---|---|
| **① 회사(사업장) 정보 관리** | 우리 회사의 기본 정보(상호·대표자·사업자번호·주소·연락처·로고·테마색)를 등록·수정 |
| **② 물류 거점(센터) 관리** | 우리 회사가 운영하는 물류 센터를 신설·변경·폐쇄 |
| **③ 물류대행(위탁) 거래** | 우리 물류를 전문 대행업체에 맡기거나, 반대로 다른 회사의 물류를 대행 수주 |

## 2. 관리 대상 정보

| 관리 단위 | 내용 |
|---|---|
| 사업장 | 회사명, 대표자, 사업자등록번호, 업태/업종, 주소, 연락처, 이메일, 사업장 구분, 사용여부, 회사 이미지, 대표 테마색 |
| 물류센터 | 사업장에 속한 창고(거점)들의 이름·주소·연락처·사용여부 |
| 위탁 관계 | 한 사업장이 다른 사업장의 물류센터에 물류 처리를 맡기는 관계 |

## 3. 업무 참여자(역할)

```
┌─────────────┐         물류 위탁 거래          ┌─────────────┐
│   의뢰자     │ ─────────────────────────────▶ │   대행업체    │
│ (화주/자사)  │   "우리 물류 맡아주세요"          │ (물류전문회사)│
│             │ ◀───────────────────────────── │             │
└─────────────┘     "수락합니다 / 거절합니다"      └─────────────┘
```

- **의뢰자(화주)**: 물류 운영을 맡기고 싶은 회사
- **대행업체**: 물류 센터를 보유하고 대행 서비스를 제공하는 회사
- **관리자**: 회사 정보와 자사 센터를 관리하는 담당자

## 4. 업무 흐름 (전체)

```
① 사업장 선택·조회
        │
        ▼
② 사업장 정보 확인/수정          ── 회사 기본정보, 이미지, 테마색 관리
        │
        ▼
③ 물류센터 관리                  ── 자사 센터 등록/수정/삭제
        │
        ▼
④ 물류 위탁 관계 형성 (선택적)
        ├─ (화주 입장) 맡길 업체를 찾아 위탁을 요청
        └─ (대행사 입장) 받은 요청을 수락 또는 거절
```

## 5. 세부 업무 시나리오

### 업무 ① 회사(사업장) 정보 관리

```
회사 정보 조회 ─▶ 정보 수정(상호/주소/연락처/로고/테마색 등) ─▶ 저장
                  · 로고 이미지는 새로 올리거나 삭제 가능
```

- 권한이 있는 사업장 목록 중 하나를 선택해 정보를 조회한다.
- 회사명, 대표자, 주소 등 기본정보와 회사 이미지·테마색을 수정한다.
- **회사명은 반드시 입력**해야 한다.
- 사업장 구분과 사용여부는 화면에서 직접 바꿀 수 없고, 변경하려면 **고객문의를 거친다.**

### 업무 ② 물류 거점(센터) 관리

```
센터 목록 조회
   │
   ├─ 신규 센터 추가 ─────▶ 센터가 즉시 운영 가능하도록
   │                        기본 물류 거점(입고/출고/반품 구역 등)이 함께 마련됨
   │
   ├─ 기존 센터 변경 ─────▶ 단, 다른 회사가 위탁 사용 중인 센터는 함부로 중단 불가
   │
   └─ 센터 폐쇄 ──────────▶ 폐쇄 전 확인:
                            · 그 센터를 쓰는 담당자가 있는가?
                            · 위탁 거래 중인가?
                            · 보관 창고가 남아있는가?
                            → 하나라도 있으면 폐쇄 불가
   ※ 회사는 최소 한 개 이상의 센터를 반드시 운영해야 함
```

- 선택한 사업장이 운영하는 물류센터 목록을 본다.
- 자사 센터는 새로 추가하거나 이름·주소·연락처를 수정·삭제할 수 있다.
- **센터명은 반드시 입력**해야 한다.
- 외부에서 위탁받은 센터는 목록에 함께 보이지만 **수정할 수 없으며**, 거절된 위탁 센터는 비활성(회색)으로 구분 표시된다.

### 업무 ③ 물류대행(위탁) 거래 — 핵심 업무

이 업무가 본 메뉴의 **가장 중요한 업무**이며, **"신청 → 응답 → (취소)"의 거래 상태 변화**로 진행됩니다.

```
 [의뢰자(화주) 입장]                          [대행업체 입장]

 (1) 대행업체·센터 검색
     "물류 맡길 회사 찾기"
     → 신청 가능 / 신청중 / 승인 / 거절
        상태가 표시됨
        │
 (2) 위탁 의뢰 신청  ────────────────────▶ (3) 들어온 의뢰 확인
     "이 센터에 맡기고 싶어요"                   "어떤 회사가 신청했나"
     (요청 내용 작성)                              │
                                          ┌──────────┴──────────┐
                                       [수락]                 [거절]
                                          │                     │
                              "우리 센터를 쓰게 해줌"      "받지 않음"
                              주소·연락처 입력 후 확정       → 의뢰 반려
                              → 거래 성립, 물류 거점 연결          │
                                          │                     │
                                          ▼                     ▼
                                       거래 진행            (재신청 가능)
 (4) 의뢰 취소
     "아직 응답 없을 때 신청 철회"
     → 신청중 상태에서만 가능
```

- **맡기는 쪽(화주)**: 물류를 맡길 수 있는 대행업체와 그 센터를 검색하고, 원하는 센터를 선택해 요청 내용을 적어 위탁을 요청한다. 요청한 건은 진행 상태(요청 중 등)로 추적되며, 확정 시 내 센터목록에 위탁센터로 반영된다(거절 시 회색 비활성).
- **맡는 쪽(대행사)**: 나에게 들어온 위탁 요청 목록을 확인하고 각 요청을 수락 또는 거절한다. 수락하는 경우, 운영할 센터의 **주소·주소상세·전화번호를 반드시 입력**해야 확정된다.

## 6. 위탁 거래 상태 흐름 (업무 규칙)

```
   [신청 전] ──신청──▶ [신청중] ──대행사 수락──▶ [승인 (거래 성립)]
       ▲                  │
       │ 취소(철회)         │ 대행사 거절
       │                  ▼
       └────────────── [거절됨] ──다시 신청──▶ [신청중]
```

| 상태 | 의미 | 가능한 다음 행동 |
|---|---|---|
| **신청중** | 의뢰자가 신청, 대행사 응답 대기 | 대행사: 수락/거절 · 의뢰자: 취소 |
| **승인** | 대행사가 수락 → 거래 성립 | 거래 운영 |
| **거절됨** | 대행사가 거절 | 의뢰자: 재신청 |

## 7. 핵심 업무 규칙 (Business Rules)

| # | 규칙 |
|---|---|
| BR-1 | 회사는 **최소 1개 이상의 센터를 운영**해야 한다 (전체 미사용 불가) |
| BR-2 | 같은 회사 내 **센터 이름은 중복될 수 없다** |
| BR-3 | **위탁 거래 중인 센터**는 폐쇄·중단할 수 없다 |
| BR-4 | 담당자 권한이 걸려있거나 창고가 남은 센터는 폐쇄할 수 없다 |
| BR-5 | 위탁 신청 시(대행사 수락 시) 대행 대상 센터는 **주소·주소상세·연락처가 필수** |
| BR-6 | 이미 **신청중/승인 상태인 거래는 중복 신청 불가** |
| BR-7 | **거절된 거래는 다시 신청** 가능 |
| BR-8 | 의뢰 **취소는 신청중 상태에서만** 가능 |
| BR-9 | 대행사가 **수락하면** 의뢰자가 대행사의 물류 거점을 사용할 수 있게 **자동 연결**된다 |
| BR-10 | **회사명은 필수** 입력이며, **사업장 구분·사용여부는 화면에서 변경 불가**(고객문의를 통해서만 변경) |
| BR-11 | **자사 센터는 자유롭게 관리**, **위탁받은 센터는 수정 불가** |
| BR-12 | **거절된 위탁 센터는 비활성(회색)**으로 표시해 구분한다 |
| BR-13 | 회사 이미지는 정해진 규격(**149×40**)으로 등록한다 |

## 8. 권한별 업무 범위

| 권한 | 조회·관리 가능 범위 |
|---|---|
| 회사 관리자 | 본인이 소속된 회사만 |
| 센터 관리자 | 본인이 권한을 가진 센터의 회사만 |
| 최고 관리자 | 산하 전체 회사 |
| (공통) | 단순 화주(shipper)는 관리 대상에서 제외 |

## 9. 한 줄 요약

> **내 사업장·센터 정보를 관리하면서, 동시에 다른 물류대행업체에 내 물류를 위탁 신청하고(의뢰자), 반대로 받은 의뢰를 수락/거절하는(대행사) 양방향 3PL 거래 관계를 "신청→승인/거절→취소" 상태 변화로 관리하는 기준정보 업무.**
