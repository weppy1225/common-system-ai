입고처리

ERP로부터 수신된 입고 정보를 기반으로 SKU를 생성하고, 라벨을 출력한 후 최종 입고를 확정하는 핵심 프로세스입니다.

소스 자동 생성 · InboundProcessing.vue + InboundProcessingController.java 커밋 a3f8b2c · 2026-04-21

개요

Route /inbound/processing API Base /api/inbound

입고처리는 입고예정 화면에서 등록된 입고건을 실제 재고로 확정하는 단계입니다. SKU 생성 → 라벨 출력 → 입고처리 순서로 진행됩니다.

상태 흐름

예정 SCHEDULED → 처리중 PROCESSING → 확정 CONFIRMED 또는 강제확정 또는 취소

검색 조건

필드선택 옵션
사업장ERP 수신 (자동)
입고유형 DOM국내입고 IMP수입입고 CEN센터입고 OUT외주센터입고 ETC기타입고
입고상태예정 · 처리중 · 확정

주요 기능

입고처리 화면은 T-Section(상단 헤더)과 B-Section(하단 품목 그리드)으로 구성됩니다.

화면 구성

1 2 3 4
1
입고지시서 출력
POST /api/inbound/print-instructions
권한 ADMIN WAREHOUSE_MANAGER

선택된 입고건의 지시서를 PDF로 출력합니다. 그리드의 체크박스를 체크 후 입고지시서 버튼을 클릭하세요.

2
강제확정
POST /api/inbound/force-confirm
권한ADMIN (전용) 트랜잭션@Transactional

입고처리가 완료되지 못한 경우 수동으로 확정 처리합니다. 상태가 PROCESSING 인 경우에만 가능하며, 완료 후 상태는 "확정"이 아닌 "강제확정"으로 별도 기록됩니다.

🚫 전제조건 위반 시
에러 코드 INBOUND_FORCE_CONFIRM_INVALID_STATUS — "강제확정은 처리중 상태에서만 가능합니다"
3
SKU 생성 / 생성취소
POST /api/inbound/sku/generate
권한WAREHOUSE_WORKER 전제조건상태 = SCHEDULED 또는 PROCESSING

입고예정 정보를 바탕으로 품목의 SKU를 생성하거나 취소합니다. 생성 후 라벨 발행 가능 상태로 전환됩니다.

4
처리 (입고 확정)
POST /api/inbound/confirm
권한WAREHOUSE_WORKER 트랜잭션@Transactional
필수 입력 inboundNo (1~20자) · processDate (LocalDate)

라벨 발행이 완료된 품목을 입고 확정합니다. 처리일자 지정이 가능합니다.

⚠ 제약 조건
라벨 발행이 된 품목에 한해서만 입고처리 가능합니다.
위반 시: INBOUND_LABEL_REQUIRED

입고 예정 취소

POST /api/inbound/cancel
권한WAREHOUSE_MANAGER 전제조건상태 = SCHEDULED

입고 유형별 취소 규칙

입고 유형취소 가능ERP 연동
DOM 국내입고✓ 가능예정 취소 결과 자동 전송
IMP 수입입고✓ 가능—
CEN 센터입고✗ 불가—
OUT 외주센터입고✓ 가능—
ETC 기타입고✓ 가능—
🚫 센터입고 취소 불가
센터입고는 취소할 수 없습니다. 에러 코드: INBOUND_CENTER_CANCEL_FORBIDDEN
💡 Tip
기입고 수량 클릭 시 입고처리 내역 팝업에서 개별 품목 단위로도 처리 취소가 가능합니다.
