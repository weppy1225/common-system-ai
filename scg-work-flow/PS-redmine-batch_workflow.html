# 구분 단계명 처리 내용 비고
1 트리거 슬래시 명령 실행 사용자가 /PS-redmine-batch 입력 Claude Code 슬래시 커맨드
2 정보 수집 WBS 엑셀 파일 경로 입력 개발자별 일감이 정의된 WBS 엑셀 파일 경로 입력
예: PS_112_WBS개발자별_반다이남코.xlsx 		산출물 폴더 내 위치
3 처리 Redmine 연결 정보 로드 설정 파일에서 Redmine URL 및 API Key 읽기
예: .claude/config/redmine.json 		Redmine REST API 인증용
4 처리 엑셀 파싱 WBS 엑셀에서 개발자별 일감 목록 추출
항목: 개발자명 · 일감명 · 버전 · 우선순위 · 시작일 · 종료일 		파싱 오류 시 오류 내용 출력 후 재입력 요청
5 처리 등록 예정 목록 미리보기 추출된 일감 목록을 테이블 형태로 출력하여
사용자가 등록 전 내용 확인 후 진행 여부 결정 		사용자 확인 단계 (취소 가능)
6 반복 처리 Redmine 일감 등록 (반복) 일감 1건씩 Redmine API 호출
POST /issues.json
담당자(assigned_to) · 버전(fixed_version) · 제목(subject) 포함 		전체 일감 수만큼 반복
7 반복 처리 등록 결과 확인 API 응답에서 issue_id 수신 → 성공 처리
오류 응답 시 실패 목록에 추가 후 다음 일감 계속 처리 		실패해도 전체 중단하지 않고 계속 진행
8 완료 보고 전체 결과 보고 성공 N건 / 실패 N건 요약 출력
실패 건에 대해 원인 및 재등록 방법 안내
WBS 엑셀 입력 구조
컬럼 항목명 예시 Redmine 매핑 필드
A 개발자명 홍길동 assigned_to_id
B 일감명 (제목) 입고예정 목록화면 개발 subject
C 버전 (스프린트) Sprint-01 fixed_version_id
D 우선순위 보통 priority_id
E 시작일 2026-06-01 start_date
F 종료일 2026-06-07 due_date
G 예상시간 8 estimated_hours

파일명 규칙: PS_112_WBS개발자별_{업체명}.xlsx

입출력 요약
구분내용
입력 WBS 엑셀 파일: PS_112_WBS개발자별_{업체명}.xlsx
Redmine 연결 정보: URL · API Key (.claude/config/redmine.json)
출력 Redmine에 개발자별 일감 일괄 등록
등록 결과 요약: 성공 N건 / 실패 N건 · 실패 건 목록
