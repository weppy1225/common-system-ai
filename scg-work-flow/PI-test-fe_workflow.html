단계별 상세
#구분단계명처리 내용비고
1트리거슬래시 명령 실행 사용자가 /PI-test-fe {메뉴코드} 입력
예: /PI-test-fe mdpd01		 BE 서버 없이 FE 단독 테스트
2처리Mockoon JSON 파일 스캔 mock/mockoon/{menu}.json 경로 탐색 SD-api 완료 후 생성된 파일
3처리Mockoon 서버 기동 확인 localhost:3000 헬스체크
미응답 시 Mockoon CLI 자동 기동:
npx @mockoon/cli start --data {menu}.json 		BE 서버 대신 목업 API 응답
4테스트목록 조회 화면 테스트 FE에서 Mockoon으로 목록 조회 API 호출
그리드에 더미 데이터 정상 표시 확인
검색 조건 변경 후 재조회 동작 확인
5테스트등록 팝업 테스트 등록 팝업 오픈 → 필드 입력 → 저장
Mockoon의 POST 응답 수신 확인
팝업 닫힘 + 목록 새로고침 동작 확인
6테스트수정 팝업 테스트 행 클릭 → 수정 팝업 오픈
Mockoon의 단건 조회 응답으로 폼 채움 확인
수정 저장 → PUT 응답 확인
7완료 보고결과 보고 테스트 항목별 통과/실패 + 화면 이상 항목 출력
입출력 요약
구분내용
입력 mock/mockoon/{menu}.json — Mockoon 목업 서버 설정
FE 소스코드 ({menu}.vue, {menu}Edt.vue)
출력 FE 단독 테스트 결과: 목록조회 / 등록 / 수정 통과 여부
