#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /PI-test-be {메뉴코드} 입력
예: /PI-test-be mdpd01		 PI-be-comp 완료 후 실행
2 처리 Bruno 파일 자동 스캔 bruno/{group}/{menu}/ 경로 탐색:
bruno.json / search.bru / insert.bru / delete.bru 		파일 없으면 PI-be-comp 먼저 실행 안내
3 처리 BE 서버 기동 확인 bruno.json의 baseUrl로 헬스체크 요청
서버 미응답 시 기동 요청 후 중단 		로컬 개발 서버 (localhost:8080)
4 테스트 search.bru 실행 목록 조회 API 호출
응답 코드 200 / 목록 배열 반환 검증
페이징 정보 포함 여부 확인 		GET /api/{메뉴코드}/list
5 테스트 insert.bru 실행 등록 API 호출
응답 코드 200 / 등록된 PK 반환 검증
DB 실제 저장 여부 확인 		POST /api/{메뉴코드}
6 테스트 delete.bru 실행 삭제 API 호출
응답 코드 200 / 삭제 건수 반환 검증
insert에서 등록한 데이터 대상 		DELETE /api/{메뉴코드}/{id}
7 완료 보고 전체 결과 보고 테스트 파일별 통과 / 실패 결과 출력
실패 시: HTTP 상태코드 + 응답 바디 + 오류 원인 분석

baseUrl, 인증 토큰 등 환경 변수 정의
