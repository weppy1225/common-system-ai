#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /PI-be-all {메뉴코드} 입력
예: /PI-be-all mdpd01		 PI-be-mapper → dao → comp 순차 실행
2 처리 설계 문서 자동 스캔 ① DEV_DOC/.../80-spec/{메뉴코드}/api.md — API 설계서
② DEV_DOC/.../80-spec/{메뉴코드}/db.md — DB 설계서 		파일 없으면 오류 출력 후 중단
3 생성 ① Mapper 레이어 생성 DB 설계서 기반 MyBatis XML Mapper 생성
{menu}Mapper.java — Mapper 인터페이스
ZTEST_{menu}Mapper.java — Mapper 단위 테스트 		SQL SELECT / INSERT / UPDATE / DELETE
4 생성 ② Dao 레이어 생성 API 설계서 + Mapper 소스 기반 Dao 생성
{menu}Dao.java — Dao 인터페이스 + 구현체
ZTEST_{menu}Dao.java — Dao 단위 테스트 		Mapper 레이어 완료 후 실행
5 생성 ③ Comp·Controller 레이어 생성 API 설계서 + Dao 소스 기반 비즈니스 레이어 생성
{menu}Search.java — 검색 조건 VO
{menu}VO.java — 데이터 VO
{menu}Response.java — API 응답 모델
{menu}CompUtil.java — 공통 유틸
{menu}TxComp.java — 트랜잭션 처리
{menu}Comp.java — 비즈니스 로직
{menu}Controller.java — REST 컨트롤러
ZTEST_{menu}CompUtil / Comp / Controller.java — 테스트 		Controller는 Comp 없으면 컴파일 불가
→ 함께 생성
6 생성 ④ Bruno 파일 생성 API 테스트용 Bruno 파일 생성:
bruno/{group}/{menu}/bruno.json — 컬렉션 설정
search.bru — 목록 조회 요청
insert.bru — 등록 요청
delete.bru — 삭제 요청 		Bruno CLI / GUI에서 실행
7 테스트 Bruno 테스트 실행 생성된 Bruno 파일로 API 통신 자동 검증
각 엔드포인트 응답 코드·데이터 구조 확인 		테스트 실패 시 원인 분석 후 소스 수정
8 완료 보고 결과 보고 생성된 전체 파일 목록 출력
Bruno 테스트 결과 요약 (성공 / 실패 건수)
