단계별 상세
#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /SD-db {메뉴코드} 입력
예: /SD-db mdpd01		 Claude Code 슬래시 커맨드
2 처리 설계 문서 자동 스캔 메뉴코드 기반으로 3가지 파일 자동 탐색:
① dist/{메뉴코드}/ui.md — 화면요건정리
② dist/{메뉴코드}/wireframe.html — 화면 설계서
③ dist/{메뉴코드}/mock-data.js — 화면 더미 데이터 		파일 없으면 오류 출력 후 중단
3 처리 실제 DB 정보 조회 개발 DB에 접속하여 기존 테이블 구조 확인
INFORMATION_SCHEMA.COLUMNS 조회
신규 테이블인지 기존 테이블 변경인지 판단 		DB 연결 정보: .claude/config/db.json
4 처리 설계 분석 화면 설계 문서와 실제 DB를 비교하여
필요한 변경 사항 도출:
· 신규 테이블 생성 (CREATE TABLE)
· 기존 테이블 컬럼 추가 (ALTER TABLE ADD COLUMN)
· 컬럼 속성 변경 (ALTER TABLE MODIFY COLUMN)
5 생성 db.md 생성 DB 설계서 마크다운 문서 생성
테이블명·컬럼명·타입·PK/FK·설명 포함
저장 경로:
DEV_DOC/ai-docs/20-backend/80-spec/{메뉴코드}/db.md 		wms-bnk-be 저장소 내 위치
6 생성 Flyway SQL 생성 변경 사항별 Flyway 마이그레이션 SQL 파일 생성
파일명 규칙: V{N}__{메뉴코드}_{테이블명}_{작업}.sql
저장 경로:
src/main/resource/db/migration/ 		버전 번호 N은 기존 파일 기준 자동 채번
7 완료 보고 결과 보고 생성된 파일 목록 출력:
db.md 경로 및 주요 테이블 목록
.sql 파일 목록 및 포함된 DDL 요약
출력 파일 경로 예시
파일 유형경로 (예: 메뉴코드 mdpd01)내용
DB 설계서 wms-bnk-be/DEV_DOC/ai-docs/20-backend/80-spec/mdpd01/db.md 테이블·컬럼 정의, PK/FK, 인덱스 설명
Flyway SQL wms-bnk-be/src/main/resource/db/migration/V1__mdpd01_mdm_prod_add_col.sql 기존 테이블에 컬럼 추가 (ALTER TABLE)
Flyway SQL wms-bnk-be/src/main/resource/db/migration/V2__iwrq01_wms_inbiz_h_create.sql 신규 테이블 생성 (CREATE TABLE)

Flyway 버전 번호(V{N})는 기존 migration 폴더 내 파일 기준으로 자동 채번

입출력 요약
구분내용
입력 dist/{메뉴코드}/ui.md — 화면요건정리
dist/{메뉴코드}/wireframe.html — 화면 설계서
dist/{메뉴코드}/mock-data.js — 화면 더미 데이터
실제 DB 테이블 구조 (INFORMATION_SCHEMA)
출력 DEV_DOC/.../80-spec/{메뉴코드}/db.md — DB 설계서
src/.../db/migration/V{N}__{메뉴코드}_{설명}.sql — Flyway 마이그레이션 SQL
