단계별 상세
#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /PI-be-mapper {메뉴코드} 입력
예: /PI-be-mapper mdpd01		 단독 실행 또는 PI-be-all 내부 호출
2 처리 설계 문서 자동 스캔 ① DEV_DOC/.../80-spec/{메뉴코드}/api.md — API 설계서
② DEV_DOC/.../80-spec/{메뉴코드}/db.md — DB 설계서 		파일 없으면 오류 출력 후 중단
3 처리 SQL 목록 도출 API 명세 + DB 설계를 분석하여 필요한 SQL 구문 결정:
SELECT 목록 조회 / 단건 조회
INSERT 단건 등록 / 배치 등록
UPDATE 단건 수정
DELETE 단건 삭제 / 논리 삭제 		API 엔드포인트 수 기준으로 SQL ID 결정
4 생성 {menu}Mapper.java 생성 MyBatis Mapper 인터페이스 생성
각 SQL에 대응하는 메서드 선언
@Mapper 어노테이션 + 파라미터 타입 매핑 		패키지: src/{그룹}/{메뉴코드}/
5 생성 {menu}Mapper.xml 생성 MyBatis SQL XML 파일 생성
각 SQL ID 별 쿼리 작성
검색 조건 동적 SQL (<if>, <where>, <foreach>)
resultMap으로 컬럼-필드 매핑 정의 		경로: resources/mapper/{그룹}/{메뉴코드}/
6 생성 ZTEST_{menu}Mapper.java 생성 Mapper 인터페이스 단위 테스트 클래스 생성
각 SQL 메서드에 대한 테스트 케이스 작성
테스트 데이터 setup / teardown 포함 		JUnit5 + Spring Boot Test
7 테스트 단위 테스트 실행 ZTEST_{menu}Mapper.java 실행
실제 개발 DB와 MyBatis 연동 검증
SQL 문법 오류 / 컬럼 매핑 오류 확인 		테스트 실패 시 SQL 수정 후 재실행
8 완료 보고 결과 보고 생성된 파일 3개 목록 출력
테스트 통과 / 실패 결과 요약
생성 파일 상세
파일명유형주요 내용경로
{menu}Mapper.java Mapper 인터페이스 SQL 메서드 선언
selectList(), selectOne()
insert(), update(), delete() 		src/{그룹}/{메뉴코드}/
{menu}Mapper.xml MyBatis SQL XML 동적 SQL (<if>, <where>, <foreach>)
resultMap 컬럼-필드 매핑
페이징 처리 SQL 		resources/mapper/{그룹}/{메뉴코드}/
ZTEST_{menu}Mapper.java 단위 테스트 각 SQL 메서드 테스트 케이스
실제 DB 연동 검증
테스트 데이터 setup / teardown 		test/.../{그룹}/{메뉴코드}/
입출력 요약
구분내용
입력 DEV_DOC/.../80-spec/{메뉴코드}/api.md — API 설계서
DEV_DOC/.../80-spec/{메뉴코드}/db.md — DB 설계서
출력 {menu}Mapper.java — MyBatis Mapper 인터페이스
{menu}Mapper.xml — MyBatis SQL XML
ZTEST_{menu}Mapper.java — Mapper 단위 테스트
