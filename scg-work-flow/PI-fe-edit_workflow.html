단계별 상세
#구분단계명처리 내용비고
1트리거슬래시 명령 실행 사용자가 /PI-fe-edit {메뉴코드} 입력
예: /PI-fe-edit mdpd01		 단독 실행 또는 PI-fe-all 내부 호출
2처리설계 문서 자동 스캔 ① api.md — 등록·수정·단건조회 API 스펙
② ui.md — 팝업 입력 항목 정의
③ wireframe.html — 팝업 화면 레이아웃
④ mock-data.js — 더미 데이터 		파일 없으면 오류 출력 후 중단
3처리폼 필드 구조 분석 팝업 입력 항목 목록 추출
필수 / 선택 항목 구분
컴포넌트 유형 결정 (Input / Select / DatePicker 등)
4생성LayerPopup 래퍼 생성 팝업 열기(open()) / 닫기(close()) 이벤트
부모 컴포넌트에서 emit으로 제어
팝업 헤더 타이틀 (등록 / 수정 모드 분기) 		공통 LayerPopup 컴포넌트 재사용
5생성폼 필드 구성 항목별 컴포넌트 배치:
Input — 텍스트 입력
Select — 코드 선택
DatePicker — 날짜 입력
Textarea — 비고 등 장문 입력
필수 항목 * 표시 + 유효성 검증 		ui.md 등록 항목 기반
6생성등록·수정 모드 전환 PK 값 유무로 모드 자동 결정
수정 모드: 단건 조회 API 호출 후 폼 데이터 채움
PK 필드 readonly 처리 		동일 컴포넌트 재사용
7생성API 저장 연동 등록 모드: POST /api/{메뉴코드}
수정 모드: PUT /api/{메뉴코드}/{id}
저장 성공 시 팝업 닫기 + 부모 목록 새로고침 emit 		api.md 등록·수정 스펙 기반
8완료 보고결과 보고 생성된 {menu}Edt.vue 경로 출력
등록 / 수정 모드 동작
등록 모드 (PK 없음)
  • 헤더 타이틀: "○○ 등록"
  • 폼 초기값으로 빈 객체 세팅
  • 저장 버튼 클릭 → POST 호출
  • 성공 시 팝업 닫기 + 목록 새로고침
수정 모드 (PK 있음)
  • 헤더 타이틀: "○○ 수정"
  • 단건 조회 API 호출 → 폼 데이터 채움
  • PK 필드 readonly 처리
  • 저장 버튼 클릭 → PUT 호출
  • 성공 시 팝업 닫기 + 목록 새로고침
입출력 요약
구분내용
입력 api.md / ui.md / wireframe.html / mock-data.js
출력 src/views/{그룹}/{메뉴코드}/{menu}Edt.vue — 등록·수정 팝업 (LayerPopup + 폼)
