#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /PI-be-excel {메뉴코드} 입력
예: /PI-be-excel mdpd01		 PI-be-comp 완료 후 실행
2 처리 기존 소스 자동 스캔 ① api.md — 엑셀 업로드 API 스펙 확인
② {menu}Comp.java — 등록 비즈니스 로직 파악
③ {menu}Dao.java — 등록 Dao 메서드 파악
④ {menu}Mapper.java — INSERT SQL 파악 		파일 없으면 오류 출력 후 중단
3 처리 엑셀 업로드 스펙 분석 엑셀 파일의 헤더 컬럼 구성 파악
각 컬럼별 유효성 검증 규칙 도출
(필수값 / 데이터 타입 / 코드 범위 등)
기존 Comp의 등록 로직 재활용 방식 결정
4 생성 {menu}X01ExcelUploadController.java 생성 @PostMapping("/excel") 엔드포인트
MultipartFile 수신 → Apache POI 파싱
행별 유효성 검증 후 Comp 등록 메서드 호출
성공 건수 / 실패 건수 + 오류 행 목록 응답 		네이밍 규칙: {MENU}X01
5 생성 ZTEST_{menu}X01*.java 생성 엑셀 업로드 컨트롤러 단위 테스트
정상 파일 / 유효성 오류 파일 / 빈 파일 케이스 포함
MockMultipartFile 활용 		JUnit5 + MockMvc
6 테스트 단위 테스트 실행 Mock 엑셀 파일로 업로드 흐름 검증
파싱 오류 / 유효성 오류 시 응답 구조 확인 		실패 시 소스 수정 후 재실행
7 완료 보고 결과 보고 생성된 파일 2개 목록 + 테스트 결과 요약
