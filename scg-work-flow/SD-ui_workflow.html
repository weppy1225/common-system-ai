단계별 상세
#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /SD-ui {메뉴코드} 입력
예: /SD-ui mdpd01		 Claude Code 슬래시 커맨드
2 처리 입력 파일 자동 스캔 메뉴코드 기반으로 3가지 리소스 자동 탐색:
① {메뉴코드}/ui.md — 화면요건정리 (메인 입력)
② common/wms-ui.css, wms-common.js — 공통 리소스
③ common/_template/base.html + 그리드 유형별 템플릿 		ui.md 없으면 오류 출력 후 중단
3 처리 UI 유형 결정 ui.md의 화면 구성 정보를 파악하여
그리드 레이아웃 유형 결정:
grid1 / grid2-tb / grid2-lr / grid3-lrt / grid3-tlb 		유형별 템플릿 html 선택
4 생성 wireframe.html 생성 base.html + 선택된 그리드 유형 템플릿을 기반으로
ui.md의 검색 조건·컬럼·업무규칙을 적용하여
{메뉴코드}/wireframe.html 생성 		공통 UI 규칙 (.claude/rules) 자동 적용
5 생성 mock-data.js 생성 그리드별 더미 데이터 10건 이상 생성
const {MENUCODE}_DATA = { ... } 형태
{메뉴코드}/mock-data.js 저장 		wireframe.html에서 <script src>로 로드
6 처리 index.html 메뉴 등록 dist/index.html의 좌측 메뉴 트리에
해당 메뉴코드 항목 자동 추가 		이미 등록된 경우 건너뜀
7 완료 보고 결과 보고 생성된 파일 목록 출력:
{메뉴코드}/wireframe.html
{메뉴코드}/mock-data.js

입출력 요약
구분내용
입력 dist/{메뉴코드}/ui.md — 화면요건정리 문서 (메인 입력)
dist/common/wms-ui.css, wms-common.js — 공통 리소스
dist/common/_template/base.html — 공통 레이아웃 템플릿
dist/common/_template/{grid유형}.html — 그리드 유형별 템플릿
출력 dist/{메뉴코드}/wireframe.html — 프로토타입 화면설계서
dist/{메뉴코드}/mock-data.js — 그리드 더미 데이터
dist/index.html — 좌측 메뉴 트리 항목 추가
