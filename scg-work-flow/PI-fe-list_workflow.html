단계별 상세
#구분단계명처리 내용비고
1트리거슬래시 명령 실행 사용자가 /PI-fe-list {메뉴코드} 입력
예: /PI-fe-list mdpd01		 단독 실행 또는 PI-fe-all 내부 호출
2처리설계 문서 자동 스캔 ① api.md — 목록 조회 API 스펙
② ui.md — 검색 조건 항목 정의
③ wireframe.html — 화면 레이아웃
④ mock-data.js — 더미 데이터 		파일 없으면 오류 출력 후 중단
3처리화면 구조 분석 ui.md와 wireframe.html에서
검색 조건 항목 목록 추출
그리드 컬럼 구성 (컬럼명 / 타입 / 너비) 추출
4생성SearchSection 생성 검색 조건 입력 영역 구현:
Input / Select / DatePicker 컴포넌트 배치
검색 / 초기화 버튼 + 엔터키 검색 이벤트 		ui.md 검색 조건 항목 기반
5생성그리드 목록 생성 그리드 컴포넌트 + 컬럼 정의 배열
체크박스 선택 / 행 클릭 이벤트
페이징 컴포넌트 (페이지 번호 / 페이지 크기) 		wireframe.html 컬럼 기반
6생성API 조회 연동 axios.get(/api/{메뉴코드}/list) 호출
검색 조건 → QueryParam 매핑
응답 데이터 → 그리드 바인딩 		api.md 목록 조회 스펙 기반
7완료 보고결과 보고 생성된 {menu}.vue 경로 출력
{menu}.vue 구성 섹션
SearchSection — 검색 조건 영역
  • Input / Select / DatePicker 컴포넌트
  • 검색 / 초기화 버튼
  • 엔터키 이벤트 바인딩
  • 조건 초기화 메서드
GridSection — 결과 목록 영역
  • 그리드 컴포넌트 + columns 정의 배열
  • 체크박스 다중 선택
  • 행 클릭 → 팝업 오픈 이벤트
  • 페이징 컴포넌트
API 연동
  • fetchList() — 검색 조건으로 목록 조회
  • onMounted 초기 조회 호출
  • 응답 데이터 → gridData 바인딩
상태 관리 (data / ref)
  • searchParam — 검색 조건 객체
  • gridData — 그리드 데이터 배열
  • pagination — 페이지 정보
  • selectedRows — 선택된 행 목록
입출력 요약
구분내용
입력 api.md / ui.md / wireframe.html / mock-data.js
출력 src/views/{그룹}/{메뉴코드}/{menu}.vue — 검색 조건 + 그리드 목록 화면
