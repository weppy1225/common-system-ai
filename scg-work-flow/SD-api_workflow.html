단계별 상세
#구분단계명처리 내용비고
1 트리거 슬래시 명령 실행 사용자가 /SD-api {메뉴코드} 입력
예: /SD-api mdpd01		 Claude Code 슬래시 커맨드
2 처리 설계 문서 자동 스캔 메뉴코드 기반으로 4개 파일 자동 탐색:
① DEV_DOC/.../80-spec/{메뉴코드}/db.md — DB 설계서
② dist/{메뉴코드}/ui.md — 화면요건정리
③ dist/{메뉴코드}/wireframe.html — 화면 설계서
④ dist/{메뉴코드}/mock-data.js — 더미 데이터 		파일 없으면 오류 출력 후 중단
3 처리 API 목록 도출 화면 기능과 DB 구조를 분석하여 필요한 API 결정:
· GET 목록 조회 / 단건 조회
· POST 등록
· PUT 수정
· DELETE 삭제
· POST 엑셀 업로드 (해당 시) 		화면 버튼·그리드 컬럼 기준으로 도출
4 생성 api.md 생성 각 API 별 명세 문서 작성:
엔드포인트 URL · HTTP 메서드
Request (PathVariable / QueryParam / Body)
Response (성공 / 오류 스펙)
저장: DEV_DOC/.../80-spec/{메뉴코드}/api.md 		wms-bnk-be 저장소 내 위치
5 생성 Mockoon JSON 생성 FE 단독 테스트를 위한 목업 서버 설정 파일 생성
api.md 명세 기반으로 각 API의 목업 응답 데이터 포함
저장: mock/mockoon/{메뉴코드}.json 		Mockoon CLI / GUI에서 임포트하여 사용
6 완료 보고 결과 보고 생성된 파일 목록 출력:
api.md — API 개수 및 엔드포인트 목록 요약
{메뉴코드}.json — Mockoon 라우트 수
API 엔드포인트 패턴
메서드URL 패턴기능RequestResponse
GET /api/{메뉴코드}/list 목록 조회 QueryParam (검색 조건) 목록 배열 + 페이징 정보
GET /api/{메뉴코드}/{id} 단건 조회 PathVariable (PK) 단건 객체
POST /api/{메뉴코드} 등록 Body (등록 데이터) 등록된 PK 또는 전체 객체
PUT /api/{메뉴코드}/{id} 수정 PathVariable + Body 수정된 객체
DELETE /api/{메뉴코드}/{id} 삭제 PathVariable (PK) 삭제 건수
POST /api/{메뉴코드}/excel 엑셀 업로드 MultipartFile 등록 성공 건수
입출력 요약
구분내용
입력 DEV_DOC/.../80-spec/{메뉴코드}/db.md — DB 설계서
dist/{메뉴코드}/ui.md — 화면요건정리
dist/{메뉴코드}/wireframe.html — 화면 설계서
dist/{메뉴코드}/mock-data.js — 더미 데이터
출력 DEV_DOC/.../80-spec/{메뉴코드}/api.md — API 명세서
mock/mockoon/{메뉴코드}.json — FE 테스트용 Mockoon 파일
