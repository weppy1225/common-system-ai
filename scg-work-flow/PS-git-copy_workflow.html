# 구분 단계명 처리 내용 비고
1 트리거 슬래시 명령 실행 사용자가 /PS-git-copy 입력 Claude Code 슬래시 커맨드
2 정보 수집 업체코드 입력 사용자에게 업체코드를 요청 후 수집
예: bnk		 저장소 이름에 소문자로 사용
3 정보 수집 업체명 입력 사용자에게 업체명을 요청 후 수집
예: 반다이남코		 완료 보고 시 표시용
4 처리 GitHub Token 로드 .claude/config/github.json에서 PAT 읽기 GitHub API 인증용
5 처리 신규 저장소 이름 결정 업체코드를 조합하여 3개 저장소명 생성:
wms-{업체코드}-doc
wms-{업체코드}-be
wms-{업체코드}-fe 		중복 여부 사전 확인
6 반복 처리 doc 저장소 생성 cloud-wms-doc 템플릿으로부터
wms-{업체코드}-doc 저장소 생성
GitHub API: POST /repos/{template}/generate 		입력: ZinCoAtoZ/cloud-wms-doc
7 반복 처리 be 저장소 생성 cloud-wms-be 템플릿으로부터
wms-{업체코드}-be 저장소 생성 		입력: ZinCoAtoZ/cloud-wms-be
8 반복 처리 fe 저장소 생성 cloud-wms-fe 템플릿으로부터
wms-{업체코드}-fe 저장소 생성 		입력: ZinCoAtoZ/cloud-wms-fe
9 완료 보고 결과 보고 생성된 3개 저장소 URL 목록을 출력하고 완료 메시지 표시
저장소 매핑
구분 역할 템플릿 저장소 (입력) 신규 저장소 (출력)
doc 문서 / 화면설계 ZinCoAtoZ/cloud-wms-doc → ZinCoAtoZ/wms-{업체코드}-doc
be 백엔드 소스 ZinCoAtoZ/cloud-wms-be → ZinCoAtoZ/wms-{업체코드}-be
fe 프론트엔드 소스 ZinCoAtoZ/cloud-wms-fe → ZinCoAtoZ/wms-{업체코드}-fe

예시 (업체코드 bnk, 업체명 반다이남코): wms-bnk-doc · wms-bnk-be · wms-bnk-fe

입출력 요약
구분내용
입력 업체코드 (사용자 입력) — 예: bnk
업체명 (사용자 입력) — 예: 반다이남코
템플릿 저장소 3개: cloud-wms-doc / cloud-wms-be / cloud-wms-fe
GitHub PAT: .claude/config/github.json
출력 GitHub 신규 저장소 3개:
ZinCoAtoZ/wms-{업체코드}-doc
ZinCoAtoZ/wms-{업체코드}-be
ZinCoAtoZ/wms-{업체코드}-fe
