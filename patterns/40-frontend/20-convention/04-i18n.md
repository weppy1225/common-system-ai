# i18n 다국어 처리

## 지원 언어

| 코드 | 언어 | 파일 위치 |
|---|---|---|
| `ko` | 한국어 (기본) | `src/assets/plugin/i18n/messages/ko.js` |
| `en` | 영어 | `src/assets/plugin/i18n/messages/en.js` |
| `ja` | 일본어 | `src/assets/plugin/i18n/messages/ja.js` |

브라우저 언어(`navigator.language`)를 자동 감지하여 초기 언어를 결정한다.

---

## 기본 사용법

### 템플릿

```vue
<!-- 버튼/라벨 텍스트 -->
<ZBtn skyblue>{{ $t('message.검색') }}</ZBtn>
<ZBtn gray>{{ $t('message.초기화') }}</ZBtn>

<!-- prop에 i18n 적용 -->
<SearchSection :title="$t('message.입고예정')">
<ZCell :title="$t('message.사업장')">
```

### script setup

```js
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

// 동적 메시지
const msg = t('message.저장되었습니다')

// 파라미터 치환
const msg2 = t('validations.minLength', { min: 2 })
```

## 메시지 키 구조

```js
// messages/ko.js
export default {
    // 날짜 포맷 함수
    date: ({list}) => {
        const d = list(0)
        return `${d.substr(0,4)}-${d.substr(4,2)}-${d.substr(6,2)}`
    },

    // Vuelidate 유효성 메시지
    validations: {
        required:  '은(는) 필수입니다.',
        email:     '이(가) 이메일 형식이 아닙니다.',
        minLength: '의 최소길이는 {min}입니다.',
        maxLength: '의 최대길이는 {max}입니다.',
        minValue:  '의 최솟값은 {min}입니다.',
        maxValue:  '의 최댓값은 {max}입니다.',
        integer:   '이(가) 숫자 형식이 아닙니다.',
        between:   '은(는) {min} ~ {max} 사이의 값이어야합니다.',
        sameAs:    '은(는) {otherName}와(과) 같은 값이어야합니다.',
    },

    // 화면 텍스트
    message: {
        검색:    '검색',
        초기화:  '초기화',
        저장:    '저장',
        // ...
    }
}
```

## 신규 메시지 키 추가 방법

**3개 언어 파일 모두 추가해야 한다.**

```js
// ko.js
message: {
    재고이동: '재고이동',
}

// en.js
message: {
    재고이동: 'Inventory Move',
}

// ja.js
message: {
    재고이동: '在庫移動',
}
```

### 누락된 키 처리

MissingHandler가 구현되어 있어, 번역 키가 없으면 `message.` 접두사를 제거한 원본 텍스트를 반환한다.  
**따라서 ko.js에만 추가하고 en/ja를 누락해도 한국어 텍스트로 폴백된다.** (장기적으로는 3개 모두 추가 권장)

## ZCodeSelect / ZCodeMulti와 i18n

```vue
<!-- 공통코드에서 불러온 이름을 번역하지 않음 — DB 값 그대로 표시 -->
<ZCodeSelect commCd="USE_YN" v-model="obj.useYn" />

<!-- 컴포넌트 자체 옵션 텍스트에는 i18n 적용 -->
<ZCodeSelect commCd="USE_YN" v-model="obj.useYn" :optionNm="$t('message.전체')" />
```

---

## ZSelect / ZMulti의 textTranslation prop

items의 text 필드 값을 `t()`로 번역할지 결정한다.

```vue
<!-- textTranslation=true: items의 text 필드를 $t()로 번역 -->
<ZSelect v-model="obj.type" :items="typeList" text="typeNm" val="typeCd"
         :textTranslation="true" />

<!-- textTranslation=false (기본): DB에서 온 텍스트 그대로 표시 -->
<ZSelect v-model="obj.bizSeq" :items="bizList" text="bizNm" val="bizSeq" />
```

## Vuelidate 에러 메시지와 i18n

```js
import { useI18n } from 'vue-i18n'
import { helpers, required, minLength } from '@vuelidate/validators'

const { t } = useI18n()

const rules = computed(() => ({
    contNm: {
        required: helpers.withMessage(
            () => `${t('message.거래처명')}${t('validations.required')}`,
            required
        ),
        minLength: helpers.withMessage(
            () => `${t('message.거래처명')}${t('validations.minLength', { min: 2 })}`,
            minLength(2)
        ),
    }
}))
```

## 주의사항

- **`$t()` 는 템플릿에서만** 사용. script에서는 `const { t } = useI18n()` 후 `t()` 사용.
- **코드 하드코딩 금지**: 화면에 표시되는 모든 문자열은 `$t('message.xxx')` 로 감싼다.
- **키 네이밍**: `message.한글키` 형식. 영문 키 혼용 금지.
- **공통코드 이름**: DB에서 조회한 값은 번역하지 않는다 (`textTranslation` 기본값 false).
