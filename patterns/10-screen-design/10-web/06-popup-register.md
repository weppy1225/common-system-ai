# 등록/수정 팝업 패턴 (WEB)

> 등록/수정 팝업의 **상세 구현 SSoT**다. 판단 기준(금지/필수)은 얇은 rule `.claude/rules/popup_reg.md`에 있다.

---

## 1. 전체 레이아웃

- 모달 팝업 형태로 화면 중앙에 표시한다.
- 구성: **헤더** → **폼 본문(테이블)** → **푸터(버튼 영역)**.
- 오버레이(`.modal-bg`) 배경: `rgba(0,0,0,0.45)`, `z-index: 1000`.
- 모달 컨테이너: `border: 1px solid #999; border-radius: 6px; box-shadow: 0 8px 32px rgba(0,0,0,0.25); background: #fff`.
- **팝업 너비 기준:**
  - 단순 입력 폼 (1컬럼): `440px ~ 500px`
  - 표준 등록/수정 폼 (3컬럼): `680px ~ 800px`
  - 검색 + 그리드 포함 팝업: `950px ~ 1050px`
  - 컬럼 수가 많은 폼은 가장 긴 레이블+입력 셀이 잘리지 않는 범위에서 조정한다.
- **모달 바디(`.modal-body`)에는 `overflow-x: hidden`을 적용하여 가로 스크롤을 차단한다.** 팝업 내부에 그리드가 포함될 경우, 그리드의 가로 넘침은 `.grid-wrap` 또는 `.panel` 안에서 처리하며 모달 바디 전체가 가로로 스크롤되지 않도록 한다.
- **팝업 내 그리드(`popup-grid-table`)의 컬럼 너비는 모달 폭 안에서 모든 컬럼의 헤더 텍스트와 데이터가 잘리지 않도록 배분한다.** No. 컬럼은 최대 데이터 건수의 자릿수를 고려하고(예: 15건이면 2자리), 코드성 컬럼(출하유형, 진행상태 등)은 전체 텍스트가 표시되는 너비를 확보한다. 컬럼이 많아 모달 폭에 맞지 않으면 모달 너비를 확장한다.
- **팝업 내 검색/초기화 버튼은 검색 영역 테이블 아래 중앙에 배치한다.** 팝업 좌측 상단에 별도로 배치하지 않는다.
- **팝업 내 그리드에서 데이터가 잘리는 경우, 그리드 래퍼에 `overflow-x: auto`를 CSS 클래스로 적용하여 가로 스크롤을 허용한다.** 이때 인라인 `style`로 `overflow-x:hidden`을 걸지 않는다 (CSS 클래스를 무시하게 되므로). 그리드 테이블은 `width: max-content; min-width: 100%`로 설정하여 컬럼 너비에 따라 자연스럽게 확장되도록 한다.
- **팝업 내에 상하 2개 그리드가 있는 경우, 상단 6 : 하단 4 비율(`flex: 6` / `flex: 4`)로 높이를 배분한다.** 하단 그리드는 데이터가 없을 때도 헤더와 빈 영역이 보이도록 `min-height`를 확보한다.

## 2. 헤더

- 배경색 `#304a6e` (프라이머리 컬러), 텍스트 흰색, `font-size: 13px; font-weight: 600`.
- 패딩 `8px 14px`, `border-radius: 6px 6px 0 0`.
- 좌측에 **"팝업명 [팝업코드]"** 형식으로 제목을 표시한다.
- **헤더 영역을 드래그하면 팝업 전체가 이동한다.** `.modal`에 `position: absolute`, `.modal-header`에 `cursor: move; user-select: none`을 적용한다. 팝업이 열릴 때 화면 중앙에 자동 위치하고, 헤더를 마우스로 드래그하여 자유롭게 이동할 수 있다. 닫기 버튼(✕) 클릭은 드래그로 인식하지 않는다.
- **팝업 코드는 "{메뉴코드}P{순번2자리}" 형식으로 자동 채번한다.** (예: MDFG01P01, DLVO01P01) 명세서에 팝업 코드가 명시되지 않은 경우에도 이 규칙에 따라 반드시 채번하여 헤더에 "팝업명 [팝업코드]" 형식으로 표시한다. 팝업 코드 없이 팝업명만 표시하지 않는다.
- 우측에 닫기 버튼 `✕` (`font-size: 20px; color: #fff`)을 배치한다.
- 헤더와 닫기 버튼은 `display: flex; align-items: center; justify-content: space-between`으로 정렬한다.

---

## 3. 폼 본문 (테이블 기반)

### 3-1. 테이블 구조

- 폼은 `<table>` 기반으로 구성한다. **레이블(th) + 입력(td)** 쌍을 한 행에 최대 **3세트** 배치한다.
- `table-layout: fixed; width: 100%; border-collapse: collapse`를 적용한다.
- 폼 영역 패딩: `padding: 12px 14px`.

### 3-2. 레이블 (th)

- 배경색 `#f3f4f6`, 텍스트 색상 `#374151`, `font-weight: 600; font-size: 13px`.
- 텍스트 **우측 정렬** (`text-align: right`), `padding: 6px 12px 6px 6px; white-space: nowrap`.
- 레이블 너비는 고정값 `80px ~ 100px`으로 설정한다.
- **폼 테이블의 레이블(th) 너비는 가장 긴 레이블 텍스트가 잘리지 않도록 설정한다.** 레이블이 긴 항목(예: "적용시작일자", "적용종료일자")이 포함된 경우 th 너비를 늘리거나 모달 너비를 확장하여 입력 컴포넌트(td)가 충분한 공간을 확보하도록 한다. 레이블 텍스트가 `white-space: nowrap`으로 설정되어 있으므로, th 너비가 부족하면 입력 셀을 침범하여 컴포넌트가 잘린다.
- 필수 항목은 레이블 뒤에 `*` 표시를 붙인다 (`color: #e74c3c; font-weight: bold`).
- 셀 테두리: `border: 1px solid #e5e7eb`.

### 3-3. 입력 셀 (td)

- 배경색 `#fff`, 셀 높이 `36px`, 패딩 `4px 10px`.
- 셀 테두리: `border: 1px solid #e5e7eb`.

### 3-4. 입력 컴포넌트

- 모든 입력/선택 컨트롤: `width: 100%; height: 26px; font-size: 13px; color: #111827; background: #fff`.
- 테두리: `1px solid #d1d5db; border-radius: 4px; padding: 0 8px`.
- 포커스 시: `border-color: #7aa2c8; box-shadow: 0 0 0 2px rgba(122,162,200,0.15)`.
- 읽기전용 필드: `background: #f3f4f6`으로 비활성 표시.

### 3-5. 컴포넌트 유형별 규칙

| 컴포넌트 | 규칙 |
|---|---|
| **Select** | 드롭다운, 높이 `26px`, 셀 너비 100% |
| **Input** | 텍스트 입력, 높이 `26px` |
| **Date** | 날짜 입력, `type="date"` 또는 달력 아이콘(📅) 버튼 포함 |
| **Textarea** | 비고 등 장문 입력, `colspan`으로 전체 폭 사용, 높이 `60px ~ 80px`, `resize: vertical` |
| **검색 팝업 연결** | 입력 필드 우측에 🔍 버튼 배치, 클릭 시 공통 팝업(CPCT01/CPPD01) 연결 |

> **⚠️ 팝업 내 공통 컴포넌트 CSS 스코프 주의:**
> 메인 화면용 CSS(`.search-area .input-with-popup` 등)는 팝업 내부(`#xxxModal`)에 적용되지 않는다. 팝업 안에서 `.input-with-popup`, `.btn-popup` 등 공통 클래스를 사용할 경우, 반드시 팝업 선택자 스코프(`#xxxModal .input-with-popup`)로 동일 스타일을 별도 정의해야 한다.

### 3-6. 빈 셀 처리

- 한 행에 3컬럼을 채우지 못할 경우, 남은 셀은 빈 칸으로 유지한다.
- 빈 레이블 셀: `background: #f3f4f6`, 빈 입력 셀: `background: #fff`.
- 테이블 구조(행·열 수)를 유지하여 시각적 정렬을 깨뜨리지 않는다.

### 3-7. 전체 폭 필드 (비고 등)

- 레이블 1칸 + 입력 `colspan="5"` (3컬럼 기준)로 전체 폭을 사용한다.
- Textarea 높이를 충분히 확보한다 (`min-height: 60px`).

---

## 4. 푸터 (버튼 영역)

- 폼 본문 아래 구분선 `border-top: 1px solid #ddd`.
- 패딩 `10px 14px`, 버튼 **중앙 정렬** (`display: flex; justify-content: center; gap: 8px`).
- 등록 모드: `[등록]` 버튼 1개.
- 수정 모드: `[저장]` + `[취소]` 버튼 2개.
- 주요 액션 버튼: `.btn-primary` 스타일 (`background: #304a6e; color: #fff`).
- 취소/닫기 버튼: `.btn` 기본 스타일 (`background: #fff; border: 1px solid #d1d5db`).

---

## 5. 행 배치 가이드

입력 항목을 행에 배치할 때 아래 우선순위를 따른다.

| 행 | 배치 기준 |
|---|---|
| 1행 | 식별·분류 정보 (사업장, 센터, 유형 등) |
| 2행 | 핵심 업무 정보 (일자, 거래처, 담당자 등) |
| 3행~ | 부가 정보 (문서번호, 참조 등) |
| 마지막행 | 비고 (전체 폭 Textarea) |

- 필수 항목(`*`)은 가능한 상단 행에 배치한다.
- 논리적으로 관련된 항목은 같은 행에 묶는다.

---

## 6. 등록/수정 모드 전환

- 동일한 팝업 HTML을 등록·수정 양쪽에서 재사용한다.
- 모드 구분: 헤더 제목 변경 (`"○○ 등록"` ↔ `"○○ 수정"`).
- 수정 모드 시 PK 필드(문서번호 등)는 `readonly` + 회색 배경 처리.
- 수정 모드 시 기존 데이터를 폼에 자동 채움.

---

## 7. 공통 팝업 연결 규칙

- 거래처/품목 입력 필드에는 반드시 공통 검색 팝업을 연결한다.
- 입력 필드는 `readonly`로 설정하여 직접 타이핑을 방지한다.
- 필드 우측에 🔍 버튼을 배치하고, 클릭 시 공통 팝업을 연다.
- 팝업에서 선택 시 번호 + 명칭 필드를 자동으로 채운다.

| 필드 유형 | 연결 팝업 |
|---|---|
| 거래처명, 거래처번호 | `CPCT01_popup.html` |
| 품목명, 품목번호 | `CPPD01_popup.html` |

> 공통 팝업 postMessage 통신 패턴 상세 → `01-common-ui.md` §3·§5.
