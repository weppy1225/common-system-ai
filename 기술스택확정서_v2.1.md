## 5. MCP (Model Context Protocol) 서버

MCP는 Claude Code가 외부 시스템과 직접 통신할 수 있게 해주는 프로토콜이다.

### 5-1. 사용 중인 MCP 서버

| MCP 서버 | 상태 | 용도 |
|---|---|---|
| Redmine MCP | 사용 중 | 이슈 조회·생성·수정, 작업시간 등록, 체크리스트 관리 |
| Playwright MCP | 사용 중 | 브라우저 자동화 (프론트엔드 통합 테스트) |

### 5-2. Redmine MCP 상세

| 도구 | 기능 |
|---|---|
| `createIssue` | 이슈 생성 |
| `updateIssue` | 이슈 수정 (진행율, 상태) |
| `createTimeEntry` | 작업시간 등록 |
| `updateTimeEntry` | 작업시간 수정 |
| `getIssues` | 이슈 목록 조회 |
| `getIssue` | 이슈 단건 조회 |
| `createChecklist` | 체크리스트 생성 |
| `getChecklistsByIssue` | 이슈별 체크리스트 조회 |
| `search` | 이슈 검색 |

### 5-3. Redmine MCP 설치 방법

#### 1단계 — 레드마인 접속 및 API 키 발급

- 레드마인 URL: `http://168.126.28.62:81/redmine`
- 로그인 후 우측 상단 **본인 이름 클릭 → 내 계정**
- 페이지 우측 사이드바에서 **"API 액세스 키"** 항목 확인
- **"표시"** 버튼 클릭 → 키 복사
  (API 키가 없으면 "재설정" 버튼으로 새로 생성)

#### 2단계 — MCP 서버 설치

터미널에서 아래 명령어 실행 (`<본인_API_KEY>` 부분을 1단계에서 복사한 키로 교체):

```bash
claude mcp add redmine -s user \
  -e REDMINE_URL=http://168.126.28.62:81/redmine \
  -e REDMINE_API_KEY=<본인_API_KEY> \
  -e REDMINE_MCP_READ_ONLY=false \
  -- npx -y @informatik_tirol/redmine-mcp-server
```

#### 3단계 — 설치 확인

```bash
claude mcp get redmine
```

`Status: ✓ Connected` 가 출력되면 정상.

### 5-4. MCP 설정 방식

```json
// .claude/settings.json 또는 전역 설정
{
  "mcpServers": {
    "redmine": { ... }
  }
}
```

## 6. 테스트 도구

### 6-1. 백엔드 테스트 (cloud-wms-be)

| 도구 | 버전 | 용도 |
|---|---|---|
| JUnit 5 | 5.10.x | 단위 테스트 (레이어별 순차 검증) |
| Bruno | 최신 stable | REST API 수동 테스트·컬렉션 관리 |

> Bruno는 컬렉션을 `.bru` 텍스트 파일로 저장하므로 Git으로 버전 관리 가능.

### 6-2. 프론트엔드 테스트 (cloud-wms-fe)

| 도구 | 버전 | 용도 |
|---|---|---|
| Playwright | 1.32.1 | E2E 테스트 (Chromium/Firefox/WebKit) |
| Mockoon | 최신 stable | BE 없이 FE 독립 개발용 REST API 목 서버 |

**테스트 실행 방식**
```bash
npm run test:e2e     # Playwright E2E 테스트
```
