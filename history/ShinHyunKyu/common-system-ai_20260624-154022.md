### 변경 내용
1. **`.claude/rules/git-workflow.md` §3 — 하드코딩 신원 표 제거 → 포인터화**
   - 이전: "신원 구조(이 머신 기준)" 표에 `ShinHyunKyu`·`weppy1225` 등 머신별 값 하드코딩
   - 이후: 신원 3종(Claude 로그인·git commit·gh) 개념만 남기고 의미·값은 `SET_USER_USAGE.md`/`SET_USER.md` 포인터로 위임
   - 사유: 허브는 모든 워크스페이스가 같은 git 레포를 클론 → 머신·작업자별로 다른 신원 값을 커밋 파일에 박으면 다른 머신에서 틀림(레포 전체 "신원 하드코딩 금지" 원칙과 정합)

2. **`.gitignore` — `SET_USER.md` 무시 추가**
   - 이후: "토큰/인증 정보" 블록에 `SET_USER.md` 추가
   - 사유: 신원 실제 값 파일은 머신별로 달라 커밋 금지

### 신규 파일 생성
- `SET_USER_USAGE.md`
  - 작업자 신원 사용법·키 사전(값 없는 가이드, 커밋 O). 신원 3종 개념·셸 도출법(`git config user.name`/`user.email`/`gh api user`)·키 의미 사전·주의점(gh=PR 관리자면 머지 금지). `SET_ENV_USAGE.md`와 동일 철학(가이드/값 분리).
- `SET_USER.md` (`.gitignore` 대상 — 커밋 안 됨)
  - 이 머신/작업자 신원 실제 값 스냅샷(로컬 전용).

### 비고
- 값은 셸로 직접 도출이 1순위(`git config`/`gh api user`). SET_USER.md는 도출 불가 사실(gh가 PR 관리자인지 등) 메모 용도.
