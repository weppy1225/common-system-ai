# 작업 히스토리

## 공통코드 개념↔구현 양방향 링크 완성: commCdStore → common-code 역참조 추가 (SoT 중복 #7 — 대부분 기분리, 잔여 링크 보완)

### 사전 확인
- `_common-arch/common-code.md`(개념/WHY/3계층 SoT)는 이미 §3.3·§6에서 구현 상세를 `40-store/01-commCdStore.md`로 위임 중.
- `01-commCdStore.md`는 WHY·3계층을 복제하지 않는 순수 FE 구현 문서. commHCd/commDCd/commDNm 은 §2 함수 파라미터 설명으로만 등장(개념 정의 복제 아님).
- → 감사가 우려한 "개념·WHY·3계층 중복"은 실제로 거의 없음. 빠진 것은 commCdStore → common-code 역방향 링크뿐.

### 변경 내용
1. **patterns/40-frontend/40-store/01-commCdStore.md — 상단 역참조 포인터 추가**
   - 이전: 개념 SoT(common-code.md)로의 역방향 링크 없음. FE 구현 문서만 단독 존재.
   - 이후: 인트로 직후 `이 문서는 FE 스토어 구현(HOW) 전용. 개념(commHCd/DCd/DNm)·WHY·3계층·시스템별 차이 → _common-arch/common-code.md (개념 SoT)` 포인터 추가.
   - 사유: 개념 SoT(common-code) ↔ 구현 SoT(commCdStore)를 양방향으로 묶고, commCdStore가 구현 전용임을 명시해 개념 중복 유입 방지.

### 비고

- #7은 신규 위임 신설이 아니라 양방향 링크 완성(역참조 1줄) 수준.
- 검증: `check-doc-refs.py` 깨진 참조 10건(기존 baseline 동일, 증가 0), commCdStore 링크 정상. PYEXIT=1 은 기존 OMS spec-상대경로 오판 건으로 무관.
