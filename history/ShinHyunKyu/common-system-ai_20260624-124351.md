# 작업 히스토리

## run-claude.bat 실행 오류 수정 — 한글 주석 파싱 깨짐

### 증상
- 실행 시 `파일 이름, 디렉터리 이름 또는 볼륨 레이블 구문이 잘못되었습니다` 에러 발생, 프롬프트가 `C:\Users\Administrator>`(홈)에 떨어짐.
- 프로브 캡처 결과 한글 주석 단어(`포함`·`스페이스`·`명세서`)와 `ME:workspace-`(변수치환 조각)가 명령으로 새어나와 실행됨.

### 원인
- 파일이 **UTF-8 + LF** 인코딩이었고, cmd.exe가 한글 주석(`rem ===` 라인)을 콘솔 코드페이지로 잘못 디코딩하면서 줄 경계가 어긋나 주석 내용이 명령으로 실행됨.
- 이 환경은 컨텍스트마다 콘솔 코드페이지가 65001/949로 일관되지 않아, `chcp 65001` 추가나 CP949 재저장으로는 안정적으로 해결되지 않음(검증 완료).

### 변경 내용
1. **줄바꿈**
   - 이전: LF only
   - 이후: CRLF
   - 사유: Windows .bat 표준. LF only는 cmd 파싱 불안정.
2. **주석 인코딩**
   - 이전: UTF-8 한글 주석
   - 이후: ASCII(영문) 주석
   - 사유: 콘솔 코드페이지에 의존하지 않는 유일하게 확실한 해법. 코드페이지(949/65001)와 무관하게 정상 파싱.
3. **로직은 동일 유지**
   - `%~dp0` 기반 HUB 도출, 워크스페이스→PROJECT 도출, BE/FE 형제 경로, `cd /d %HUB% && claude ...` 그대로 유지.

### 검증
- claude 부분만 `cd > OUT` 프로브로 치환해 실제 실행 → cwd=허브, `--add-dir` 경로(Screenshots·common-system-be·common-system-fe) 모두 정상 해석, 한글 누출 에러 없음 확인.
