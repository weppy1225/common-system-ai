# 작업 히스토리

## 메뉴코드 형식 규칙 중복 정리: FE naming §3 → BE menu-code-rule §1 위임 (SoT 중복 #6 해소)

### 변경 내용
1. **patterns/40-frontend/20-convention/01-naming.md §3 "메뉴 코드" 위임 전환**
   - 이전: 메뉴코드 형식("영문소문자 4자 + 숫자 2자")을 BE 정본 참조 없이 독립 기술. `02-menu-code-rule.md §1`(형식 + 약어표 + 타입번호표)과 형식 부분 중복.
   - 이후: 맨 앞에 `형식·업무약어표·타입번호 정의 SoT → 30-backend/20-rule/02-menu-code-rule.md §1` 포인터 추가. 형식 정의 중복 제거. FE 고유 규칙은 유지·정리: 케이스 매핑(식별자 대문자 MDCT01 / FE 파일·라우트 소문자 mdct01 / route.meta.menuCd 대문자), FE 검증 정규식 `^[a-z]{4}\d{2}$`, `.vue` 폴더 경로, 팝업 파일명(Edt/Lbs/Dtl).
   - 사유: 메뉴코드 형식·약어·타입의 정본은 BE menu-code-rule §1. FE 는 같은 코드를 케이스·파일경로 관점으로 쓰므로 FE 고유 사실만 남기고 공유 형식은 위임. 드리프트 위험 제거 + FE 전용 정보 손실 없음.
