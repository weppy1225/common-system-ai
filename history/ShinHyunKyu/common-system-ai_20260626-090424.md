# 작업 히스토리

## SD_db / SD_db_apply 신 구조 마이그레이션 + DB 변경이력 파일 신설

마지막 보류 항목(SD_db·SD_db_apply 의 ai-docs 워크플로)을 신 구조로 재설계했다. 사전 합의: 변경이력 파일명 `00-database-deploy-history.md`(A안 — 기존 `00-tables-overview.md` 와 `00-` 공존, 알파벳상 위로 정렬), 테이블 명세는 도메인 그룹 파일 목록 행 추가 방식.

### 신규 파일 생성
- `spec/common-system/_knowledge/db-schema/00-database-deploy-history.md`
  - WMS DB 변경·반영 이력 로그. 표 헤더: `반영일시 | 메뉴코드 | 테이블/시퀀스 | 변경 내용 | 테스트반영일 | 운영반영일`
- `spec/kyochon-oms/_knowledge/db-schema/00-database-deploy-history.md`
  - OMS 동일 구조 (PostgreSQL 대상 명시)

### 변경 내용

1. **두 프로젝트 `00-tables-overview.md` 인덱스에 deploy-history 링크 추가**
   - 사유: 폴더 진입 문서에서 변경이력 파일 발견 가능하도록

2. **`.claude/skills/SD_db_apply/SKILL.md` 신 구조 재작성**
   - 이전: 입력 `db.md`, 출력 BE `DEV_DOC/ai-docs/10-database/*` 트리(change-history·90-schema/20-tables·10-architecture/sequence 등)
   - 이후: 입력 `$AI_DIR/spec/$PROJECT/{메뉴코드}/{메뉴코드}-03-data-model.md` 섹션6 DDL, 출력 허브 `spec/$PROJECT/_knowledge/db-schema/`(deploy-history·도메인그룹 목록·90-common-code·tables-overview)
   - Step 0: cd 제거 → 허브($AI_DIR 설계·문서) + BE($BE_DIR properties) 변수 명시
   - Step 6: 6-a 변경이력→deploy-history, 6-b per-table 파일생성→그룹파일 목록행 추가(컬럼명세는 psql SoT), 6-c/6-d 공통코드·overview 허브 경로
   - 사유: 커밋 577fc54 ai-docs 마이그레이션 미완분 정리 + 신 db-schema 구조(도메인그룹·change-history 부재)에 정합

3. **`.claude/skills/SD_db/SKILL.md` 신 구조 재작성**
   - 이전: 출력 BE `DEV_DOC/ai-docs/.../db.md`, 규칙문서 BE `10-database/*` 참조
   - 이후: 출력 `$AI_DIR/spec/$PROJECT/{메뉴코드}/{메뉴코드}-03-data-model.md`(SD_api 기대 경로와 일치), 규칙문서 `patterns/20-database/10-domain-and-types.md`·`20-rule/04-sequence-creation-rule.md`·`spec/$PROJECT/_knowledge/db-schema/` 참조
   - DB_MD 출력 변수 추가, 템플릿 `[ui.md](ui.md)` → `{메뉴코드}-02-ui.md`, 섹션 제목·전제조건 정리
   - 사유: SD_db 출력이 SD_api 입력(`-03-data-model.md`)과 불일치하던 문제 해소

### 검증
- 두 스킬에 `db.md`·`DEV_DOC`·`ai-docs`·`80-spec`·`10-database`·`90-schema` 잔여 0건.
- 전 라이브 문서 ai-docs 참조는 PI_be_* 의 "사용 금지" 안내문만 남음(정상).
- 링크 체커: 신규 deploy-history 인덱스 링크 정상 resolve. 잔여 깨짐은 history 로그·오탐(common-system `_meta.md` 이전위치 열, CLAUDE.md 경로 예시)뿐.

### 후속 처리 완료
- STRUCTURE.md `patterns/20-database/` 설명: `(도메인·타입·변경이력)` → `(도메인·타입·네이밍·시퀀스·SQL컨벤션)`. "변경이력"은 spec deploy-history 로 이동해 patterns 에 없으므로 실제 내용에 맞게 정정.
