# 작업 히스토리

## SET_ENV.md 생성 — 고객 이식 시 채울 환경 사실 체크리스트(SoT)

### 신규 파일 생성
- `SET_ENV.md`
  - 워크스페이스(고객 프로젝트)마다 달라지지만 폴더명에서 도출되지 않는 "환경 사실"의 단일 출처(SoT).
  - `PORTING.md §06`이 권장한 `REPOSITORY.md` 아이디어를 `SET_ENV.md`(`.env`/`.env.example` 컨벤션)로 구체화.
  - 구성: 0.사용법(보안 규칙) / 1.프로젝트 정체성(도출값 참고) / 2.배포 식별자 4종 / 3.서버·호스트·포트 / 4.DB 접속 / 5.FTP·배포 / 6.레드마인 / 7.외부연동 / 8.브랜드 표기 / 부록(점검 grep).
  - 보안: 🔒 비밀값(DB 비번·JWT·FTP 비번·토큰)은 파일에 적지 않고 BE `application-{profile}.properties`(ENC) 포인터만 둔다 — `.claude/rules/oms-security.md` NEVER 규칙 준수.
  - 비밀 아닌 환경값(컨텍스트 경로·WAR명·FTP 폴더명·레드마인 키)은 env 블록(`KEY=value`)으로 채워 커밋 가능.

### 배경
- 리브랜딩/고객 이식 조사 결과, 이름·경로 차원은 `repo-paths.md` 런타임 도출로 이미 방어됨.
- 진짜 지뢰는 폴더명으로 도출 불가한 환경 사실(DB·FTP·호스트·레드마인·외부연동)과 고정값 잔재.
- 이를 한곳에 모으는 SoT가 필요해 `SET_ENV.md` 생성.
