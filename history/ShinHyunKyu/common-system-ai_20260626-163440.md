# 작업 히스토리

## knowledgebase/ 폴더 완전 제거 + 전 참조 정리 (도메인 폴더 미승격, 단순 삭제)

세션 시작 시 `knowledgebase/` 는 `00-overview.md` + `domains/{wms,oms}/README.md`(라우터) 3개만 남은 상태였다. 사용자 지시에 따라 **승격 없이 폴더 자체를 삭제**하고, 모든 참조를 정리했다. 새 모델: **② 도메인 표준 = `.claude/rules/{system}-*` 규칙만**(별도 도메인 폴더 없음). 도메인 개념·실데이터는 기준 프로젝트의 ③(`spec/{프로젝트}/_knowledge/`)에 통합.

### 변경 내용

1. **OMS README의 고유 콘텐츠 보존**
   - 이전: `knowledgebase/domains/oms/README.md` 의 "숨은 전제"(전통 Spring·Ant WAR·PostgreSQL+SQL Server 멀티 DB·Vue3+Vite+Pinia·FE 한 레포 두 UI 시스템(`/bc` 쇼핑몰·`/be` Admin))
   - 이후: `spec/kyochon-oms/_knowledge/_meta.md` 에 "### OMS 기술 전제 (도메인 공통)" 절로 보존
   - 사유: 삭제 시 유실 방지. OMS 유일 프로젝트이므로 프로젝트 메타에 둠

2. **spec/kyochon-oms/_knowledge/_meta.md**
   - ② 도메인 표준 위치를 `knowledgebase/domains/oms/` → `.claude/rules/oms-*` 로 변경, README 행 제거, 기술 전제 절 추가

3. **spec/common-system/_knowledge/_meta.md**
   - ② 도메인 표준 위치를 `knowledgebase/domains/wms/` → `.claude/rules/wms-*` 로 변경
   - "이전 위치" 열의 죽은 `knowledgebase/...` 경로 5건 → "(구 …에서 통합)" 서술로 대체

4. **CLAUDE.md** — 3계층 표·경계 표 등 4개소에서 `knowledgebase/domains/{도메인}/` 참조 제거, ② = `.claude/rules/{system}-*` 로 정정

5. **STRUCTURE.md** — 최상위 트리 라인·역할 표 행·`## knowledgebase/` 섹션 제거. ①②③ 라벨 설명은 "## ①②③ 계층 라벨" 절로 대체(SoT=CLAUDE.md). 경계규칙 4번 갱신

6. **README.md** — "무엇이 어디에" 목록에서 `knowledgebase/` 제거

7. **.claude/rules/system-detect.md** — §0 작업분류 표·노트·§3 ② 도메인 표준 행에서 `knowledgebase/` 진입점 참조 제거 → `.claude/rules/{system}-*`

8. **spec/kyochon-oms/_knowledge/db-schema/05-interface-tables.md** — 인터페이스 컨벤션 참조를 `knowledgebase/domains/oms/` → 도메인 룰·프로젝트 인터페이스 명세로 변경

9. **scripts/gen-md-map.py** — META 영역 목록에서 `knowledgebase` 항목 제거

10. **scripts/check-doc-refs.py** — `SCAN_DIRS`·docstring·정규식 주석에서 `knowledgebase` 제거

11. **edu/ai-dev-procedure.html** — 푸터의 죽은 원본 경로(`knowledgebase/50-dev-workflow/...`) 제거

12. **edu/md-index.html** — `gen-md-map.py` 재실행으로 재생성(knowledgebase 영역 사라짐)
