# 작업 히스토리

## glossary-std(표준 약어) 를 domains/wms(tier②) → common-system glossary(tier③) 통합

### 결정·판단
"common-system = 유일 WMS 기준" 연장선. `glossary-std.md`(IW·RQ·PC·SC 업무 약어)는 스킬이 파싱하지 않는 사람 참조용이고, 이미 ③ 층에 `glossary/dictionary.yaml`(컬럼 사전)이 있으므로 같은 `glossary/` 폴더로 통합. 형태가 달라(약어표 vs 컬럼맵) YAML 병합 대신 `abbreviations.md` 로 폴더 통합.

### 변경 내용
1. **파일 이동 (git mv)** `knowledgebase/domains/wms/glossary-std.md` → `spec/common-system/_knowledge/glossary/abbreviations.md`
   - frontmatter: title "표준 약어", domain database→wms, project 추가, agent_usage instruction→reference, dictionary.yaml 상호참조 안내. 빈 제목줄 → `# 표준 약어` 헤더.
2. **`knowledgebase/domains/wms/README.md` 재작성** — "무엇을 넣나" 표가 비어, **WMS 진입점·라우터**로 전환(system-detect 룰이 참조하므로 삭제 불가). WMS 지식 위치(약어·컬럼사전·SIF컨벤션·업무흐름·스키마·install-guide)를 모두 `spec/common-system/_knowledge/` 로 라우팅하는 표 제공.
3. **`_meta.md`** — ② 행 "표준 약어 등" → "진입점 README(라우터)만", ③ 행에 "표준 약어" 추가, `_knowledge/` 구성표에 `glossary/abbreviations.md` 행 추가.

### 검증
- `python scripts/gen-md-map.py` → `edu/md-index.html` (md 192개).
- `python scripts/check-doc-refs.py` → 깨진 참조 0, 미참조 패턴 0, BOM 0 (종료코드 0).
- 라이브 `glossary-std` 잔여참조 0 (단 `_meta.md` "이전 위치" 출처열 1건 의도 보존).
- 이전 후 `knowledgebase/domains/` = `wms/README.md` + `oms/README.md` (둘 다 진입점 라우터만).
