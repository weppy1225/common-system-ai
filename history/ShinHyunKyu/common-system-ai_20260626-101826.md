# 작업 히스토리

## check-doc-refs.py 오탐 수정: 하위경로 내 'patterns/' substring 제외

### 원인
`scripts/check-doc-refs.py` 의 `RE_PATTERN_REF = patterns/[A-Za-z0-9/_.\-]+\.md` 정규식이 앵커 없이 substring 매칭하여, OMS 규칙·common-code 의 `spec/{$PROJECT}/_knowledge/patterns/...` · `knowledgebase/domains/oms/patterns/...` 같은 **하위경로 안의 'patterns/...'** 를 repo-root 참조로 오인 → 존재하지 않는 `patterns/db/...`·`patterns/be/...`·`patterns/fe/...` 로 보고. 깨진 참조 10건이 모두 이 오탐이었음.

### 변경 내용
1. **scripts/check-doc-refs.py:28 `RE_PATTERN_REF` 정규식 보강**
   - 이전: `re.compile(r"patterns/[A-Za-z0-9/_.\-]+\.md")`
   - 이후: `re.compile(r"(?<![/\w])patterns/[A-Za-z0-9/_.\-]+\.md")` — 앞에 경로구분자(/)·단어문자가 붙은 하위경로 내 'patterns/' 를 negative lookbehind 로 제외. 주석으로 사유(프로젝트 템플릿·도메인 상대경로는 repo-root patterns/ 가 아님) 명시.
   - 사유: repo-root `patterns/` 무결성만 검사해야 하는데 spec/domains 하위의 동명 경로를 오탐. lookbehind 로 토큰 경계의 repo-root 참조만 매칭.

### 검증
- 수정 후 `python scripts/check-doc-refs.py`: **PYEXIT=0, [OK] 깨진 참조 없음** (이전 10건 → 0건).
- 참조 41개 정상 집계 — genuine repo-root patterns 참조(예: be-layer-pattern.md, 01-api-naming-rule.md 등)는 그대로 검증됨(오탐만 제거, 정상 탐지 유지).
- WARN 미참조 패턴 문서 13건은 별개 항목(이번 변경과 무관).

### 비고
- 진짜 깨진 참조는 원래 0건이었고 baseline 10건은 전부 이 substring 오탐이었음이 확정됨. 이전 SoT 작업들의 "baseline 10건 유지" 판단도 옳았음(회귀 아님).
