# 작업 히스토리

## OMS 구현 패턴 파일을 knowledgebase/domains/oms/ → spec/kyochon-oms/_knowledge/ 로 이동

### 변경 내용
1. **파일 이동 (8개)**
   - 이전: `knowledgebase/domains/oms/patterns/be/`, `db/`, `fe/`, `install-guide/`
   - 이후: `spec/kyochon-oms/_knowledge/patterns/be/`, `db/`, `fe/`, `install-guide/`
   - 사유: `patterns/`는 시스템 무관 공통 패턴 전용. OMS 특화 구현 규칙은 프로젝트 지식베이스(`spec/{프로젝트}/_knowledge/`)에 위치해야 함. `knowledgebase/domains/oms/`는 도메인 개념 전용으로 정리

2. **rule 파일 참조 경로 수정 (4개 파일)**
   - `.claude/rules/oms-backend-convention.md` — be 패턴 2개 경로 수정
   - `.claude/rules/oms-db-convention.md` — db 패턴 3개 + be 채번 1개 경로 수정
   - `.claude/rules/oms-frontend-convention.md` — fe 패턴 2개 + install-guide 1개 경로 수정
   - `.claude/rules/common-code.md` — OMS commCdStore 경로 수정
   - 변경 패턴: `knowledgebase/domains/oms/patterns/*` → `spec/{$PROJECT}/_knowledge/patterns/*`

3. **knowledgebase/domains/oms/README.md 내용 정비**
   - 이전: patterns/·install-guide/ 폴더가 이 디렉토리에 있는 것처럼 기술
   - 이후: 도메인 개념 전용 역할로 재정의. 구현 패턴·기동 가이드는 spec/{프로젝트}/_knowledge/로 안내하는 표로 교체
