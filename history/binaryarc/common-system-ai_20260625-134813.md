# 작업 히스토리

## frontmatter priority 필드 정의 — 겹치는 rule 충돌 해소용

### 변경 내용
1. **`.claude/rules/rule-skill-frontmatter.md`**
   - 이전: `description`, `paths` 두 필드만 정의
   - 이후: `priority` 선택 필드 추가 (겹치는 rule이 생길 때만 사용)
   - 사유: rule 간 충돌 시 우선순위 명시 수단 마련. 허용값 `critical | high | medium | low` 기준표 및 체크리스트 항목 추가

2. **`.claude/rules/md-frontmatter.md`**
   - 이전: `priority` 필드 없음
   - 이후: 템플릿에 `priority` 선택 필드 추가 + 허용값 기준표 + MUST 규칙 명시
   - 사유: 일반 문서에서도 rule/skill 충돌 시 priority 작성 기준 동일하게 참조할 수 있도록
