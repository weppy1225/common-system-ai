# 작업 히스토리

## SHPD01 SHOP_CONT AES256 암호화 보정 등록 구현

### 변경 내용

1. **SHPD01ShopCont.java 신규 생성**
   - 이전: 없음
   - 이후: `src/main/java/be/sh7000/shpd01/bean/SHPD01ShopCont.java` 생성
   - 사유: MDM_CONT에서 SELECT한 평문 + 암호화된 byte[] 필드를 함께 담는 DTO. SQL SELECT-INSERT 방식을 Java 경유 방식으로 교체하기 위해 필요

2. **SHPD01Mapper.xml — ensureShopContFromMdmCont 제거, 3개 메서드로 교체**
   - 이전: `ensureShopContFromMdmCont` (INSERT ... SELECT 단일 SQL, 암호화 불가)
   - 이후: `existShopCont` / `selectMdmContForShopCont` / `insertShopCont` 3개로 분리
   - 사유: Java에서 암호화 처리 후 INSERT 하기 위해 SQL을 단계 분리

3. **SHPD01Mapper.java — 인터페이스 메서드 교체**
   - 이전: `ensureShopContFromMdmCont(SHPD01Prod)` → int
   - 이후: `existShopCont(Integer)` / `selectMdmContForShopCont(Integer)` / `insertShopCont(SHPD01ShopCont)` 추가, 기존 메서드 제거
   - 사유: Mapper.xml 변경에 맞춰 인터페이스 동기화

4. **SHPD01Dao.java — Dao 메서드 교체**
   - 이전: `ensureShopContFromMdmCont(SHPD01Prod)` (내부에서 syncShopContSeq 포함)
   - 이후: `existShopCont(Integer)` → boolean / `selectMdmContForShopCont(Integer)` → SHPD01ShopCont / `insertShopCont(SHPD01ShopCont)` (내부에서 syncShopContSeq 포함)
   - 사유: Mapper 변경에 맞춰 Dao 동기화. syncShopContSeq는 insertShopCont 내부로 이동

5. **SHPD01TxComp.java — 암호화 로직 private 메서드로 구현**
   - 이전: `shpd01Dao.ensureShopContFromMdmCont(prod)` 직접 호출 (insertProdTX, updateProdTX 각 1회)
   - 이후: private `ensureShopContFromMdmCont(SHPD01Prod)` 메서드로 교체. 내부에서 existShopCont 체크 → selectMdmContForShopCont → CryptoTool.encryptAES256 → insertShopCont 순 처리
   - 사유: post_no, addr, addr_dtl을 AES256 암호화하여 SHOP_CONT.bytea 컬럼에 저장

### 신규 파일 생성

- `src/main/java/be/sh7000/shpd01/bean/SHPD01ShopCont.java`
  - MDM_CONT → SHOP_CONT 보정 등록용 DTO. 평문 필드(postNo/addr/addrDtl)와 암호화 필드(postNoEncrypt/addrEncrypt/addrDtlEncrypt) 분리 보유
