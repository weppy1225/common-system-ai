1

스킬명

SD_db (DB 설계 문서 자동 작성)

실행: /SD_db {메뉴코드}

2

주요 작업 내용

화면설계 파일을 읽어 DB 변경사항(신규 테이블 / 컬럼 추가 / 공통코드 추가)을 도출하고, DDL SQL이 포함된 데이터모델 문서(-03-data-model.md)를 작성한다.

3

스킬 개요

화면설계(ui.md · wireframe)로부터 필요한 테이블·컬럼을 도출한 뒤, 기존 DB 스키마와 대조하여 변경 중심으로 간결하게 작성하는 스킬이다. 변경이 없는 기존 테이블은 목록만 기재하고 상세는 생략한다.

산출물 -03-data-model.md의 DDL SQL 섹션을 검토자가 확인 후 /SD_db_apply로 실 DB에 반영한다. 이후 /SD_api로 API 명세를 이어서 작성한다.

4

육하원칙

누가DB 설계 담당자가 Claude와 함께 (AI가 화면 분석·테이블 설계 수행)
언제화면요건 문서(ui.md, SD_310_UI 산출물)와 wireframe이 준비된 뒤
어디서AI 허브(common-system-ai) CWD에서 /SD_db {메뉴코드}
무엇을화면의 필드·상태코드·업무규칙을 분석해 테이블 구조와 DDL SQL을
어떻게기존 스키마와 대조 + 도메인 타입 규칙 적용 + 불확실 컬럼은 사용자 확인 후
왜검토·승인·DB 반영까지 한 사이클로 처리하기 위해
5

실행 순서 (7 Steps)

  • Step 1
    화면설계 파일 로드 ui.md · wireframe.html · mock-data.js 읽기 → 필요 테이블·컬럼 후보 도출
  • Step 2
    DB 규칙 문서 로드 전체 테이블 목록(00-tables-overview.md) · 도메인 타입 규칙 · 공통코드 목록 읽기
  • Step 3
    테이블 신규/기존 판정 신규목록에 없음 → DDL 작성 대상 기존목록에 있음 → 변경 없으면 생략 컬럼추가기존 테이블이나 컬럼이 없음 → ALTER 대상 BLOCKINGFE에는 있는데 DB·SQL에 없는 필드 → 의도 반드시 질문
  • Step 4
    공통코드 현황 파악 _cd 컬럼 식별 → 기존 코드그룹 확인 → 신규 코드그룹이면 INSERT SQL 작성
  • Step 5
    컬럼 도메인 타입 결정 H 테이블 seq = int4 · D 테이블 seq = bigint · 날짜 = varchar(8) YYYYMMDD · 일시 = timestamp
  • Step 6
    데이터모델 문서 작성 변경 요약 → 사용 테이블 → 신규 명세 → 컬럼 추가 → 공통코드 → UP/DOWN DDL SQL
  • Step 7
    확인 요청 → 다음 단계 안내 DDL SQL 검토 요청 → 확인 완료 시 /SD_db_apply → /SD_api 순서 안내
