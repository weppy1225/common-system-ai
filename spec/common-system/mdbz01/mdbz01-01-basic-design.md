# MDBZ01 기본설계 — 사업장

## 1. 업무 정의

사업장 메뉴(MDBZ01)는 WMS 시스템을 사용하는 기업의 사업장 기본 정보와 해당 사업장에 속하는 물류센터를 관리하는 기준정보 화면이다. 사업장 관리자는 이 화면에서 자신이 속한 사업장의 명칭, 대표자, 사업자등록번호, 주소, 연락처, 로고 이미지, 테마 색상 등 사업장 프로필 정보를 수정할 수 있다. 또한 사업장 아래에 물류센터를 추가·수정·삭제하여 물류 거점을 관리한다.

| 구분 | 내용 |
|---|---|
| 운영 형태 1 | 자사물류: 자신의 물류 인프라를 직접 보유·운영하는 형태 |
| 운영 형태 2 | 위탁물류: 물류 처리를 다른 전문 물류 대행 업체에 위탁하는 형태 |
| 위탁물류 업무 | 위탁물류를 이용하는 사업장은 이 화면에서 물류 대행 업체를 검색하고 위탁 의뢰를 신청할 수 있다. 의뢰를 받은 대행 업체는 수락 또는 거절 처리로 사업 관계를 수립한다. |
| 사업장 선택 | 관리자 권한 사용자는 자신이 접근 가능한 복수 사업장 중 수정할 사업장을 선택해 작업할 수 있다. |
| 수정 대상 제외 | 화주(납품업체)로 분류된 사업장은 이 화면의 수정 대상에서 제외된다. |

## 3. 업무 참여자 / 역할

| 참여자 | 역할 |
|---|---|
| 사업장 관리자 | 사업장 기본 정보 수정, 물류센터 추가·수정·삭제 |
| 슈퍼 관리자 | 모든 사업장 수정 가능, 신규 센터 등록 시 자동으로 권한 부여 받음 |
| 사업장 권한 사용자 | 자신이 속한 사업장만 수정 가능 |
| 센터 권한 사용자 | 자신이 접근 가능한 센터가 속한 사업장만 수정 가능 |
| 물류 대행 업체 | 위탁 의뢰를 수락하거나 거절하는 역할 |
| 의뢰 사업장(화주) | 물류 대행 업체를 검색하고 의뢰를 신청하는 역할 |

## 4. 업무 흐름 전체

```
[화면 진입]
     │
     ▼
 수정 가능한 사업장 목록 조회 (권한에 따라 목록 범위 결정)
     │
     ▼
 사업장 선택 (드롭다운)
     │
     ├─── 사업장 기본 정보 로드 (좌측 영역)
     │
     └─── 해당 사업장의 물류센터 목록 로드 (우측 영역)

[사업장 정보 수정]
     │
     ├── 정보 입력 (사업장명 필수)
     ├── 로고 이미지 업로드/삭제 (선택)
     ├── 테마색 선택 (선택)
     └── [저장] 클릭
              │
              ▼
         유효성 검사 (사업장명 필수 확인)
              │
              ▼
         사업장 정보 저장 + 이미지 처리
         (신규 이미지: 기존 삭제 후 신규 등록 / 이미지 삭제: 기존 이미지 제거)

[물류센터 관리]
     │
     ├── 센터 추가 (그리드에 행 추가 후 센터명 입력)
     ├── 센터 수정 (그리드 셀 직접 편집)
     └── 센터 삭제 (체크 후 삭제)
              │
              ▼
         [저장] 클릭
              │
              ▼
         유효성 검사 (센터명 필수)
         → 중복 센터명 검사
         → 위탁 업체 존재 시 비활성화/삭제 차단
         → 사용자 권한 존재 시 삭제 차단
         → 창고 존재 시 삭제 차단
              │
              ▼
         변경 내용 저장
              │
              ▼
         하나 이상의 센터가 사용 상태인지 최종 확인
         → 모든 센터 미사용 시 저장 차단

[위탁 업무 흐름]
     │
     ├── [물류 대행 업체 검색 팝업] 열기
     │        │
     │        ├── 사업장명/센터명/주소 조건으로 검색
     │        ├── 검색 결과 그리드에서 업체 체크
     │        ├── 요청 내용 입력
     │        └── [위탁 요청] 클릭 → 대행 의뢰 신청 저장
     │
     └── [대행의뢰신청업체 목록] 에서 수락/거절 처리
              │
              ├── 수락: 대행 관계 수립 + 해당 센터의 창고 정보 접근 권한 부여
              └── 거절: 대행 관계 거절 상태로 변경

     [의뢰 취소]
     의뢰자가 아직 처리되지 않은 의뢰를 직접 취소
```

## 5. 세부 업무 시나리오

### 시나리오 A: 사업장 기본 정보 수정

1. 담당자가 화면에 진입하면 자신의 권한 범위 내에서 수정 가능한 사업장 목록이 드롭다운에 자동 로드된다.
2. 첫 번째 사업장이 자동 선택되고 기본 정보 폼에 현재 값이 채워진다.
3. 담당자가 필요한 항목을 수정한다. 로고 이미지는 파일 선택 버튼으로 업로드할 수 있으며, 기존 이미지를 삭제할 수도 있다.
4. [저장] 클릭 시 사업장명 필수 입력 검증이 수행된다.
5. 검증 통과 시 변경된 정보가 저장되고 화면이 갱신된다.

### 시나리오 B: 물류센터 추가

1. 우측 물류센터 그리드에서 행 추가 버튼을 클릭하여 새 행을 삽입한다.
2. 센터명, 주소, 전화번호 등을 입력한다. 주소는 우편번호 아이콘 클릭으로 검색 팝업을 이용할 수 있다.
3. [저장] 클릭 시 서버에서 다음 항목을 검증한다:
   - 센터명이 입력되었는지 확인
   - 동일 사업장 내에 같은 이름의 센터가 이미 존재하는지 확인
4. 검증 통과 시 센터가 등록되며, 동시에 다음이 자동으로 처리된다:
   - 기본 창고 세트가 해당 센터에 자동 생성된다 (사전 정의된 템플릿 기반)
   - 슈퍼 관리자 권한을 가진 사용자 전원에게 해당 센터 접근 권한이 자동 부여된다

### 시나리오 C: 물류센터 수정

1. 그리드에서 수정하고 싶은 센터의 셀을 직접 편집한다.
2. 위탁 대행 업체가 사용 중인 센터는 미사용 전환이나 물류대행 여부 변경이 차단된다.
3. [저장] 클릭 시 변경 내용이 저장된다.

### 시나리오 D: 물류센터 삭제

1. 그리드에서 삭제할 센터를 선택한다.
2. 서버에서 삭제 가능 여부를 검증한다:
   - 해당 센터에 접근 권한을 가진 사용자(슈퍼 권한 제외)가 있으면 삭제 불가
   - 해당 센터를 사용 중인 위탁 대행 관계가 있으면 삭제 불가
   - 해당 센터에 창고가 존재하면 삭제 불가 (창고를 먼저 삭제해야 함)
3. 검증 통과 시 해당 센터가 비활성화되고 관련 사용자 권한 정보가 정리된다.
4. 저장 완료 후 사업장 내 사용 중인 센터가 하나도 없으면 저장 자체가 차단된다.

### 시나리오 E: 물류 대행 의뢰 신청

1. 화면에서 [물류 대행 업체 검색] 팝업을 연다.
2. 사업장명, 센터명, 주소 조건으로 위탁 가능한 업체를 검색한다.
3. 검색 결과에는 각 센터별 현재 요청 상태(신청가능 / 신청중 / 승인 / 거절)가 표시된다.
4. 원하는 업체의 센터를 체크하고, 요청 내용(메모)을 입력한 뒤 [위탁 요청] 버튼을 클릭한다.
5. 이미 신청 중이거나 승인된 센터는 중복 신청이 차단된다. 거절된 센터는 재신청이 가능하다.

### 시나리오 F: 위탁 의뢰 수락/거절 (대행 업체 측)

1. 대행 업체 담당자가 화면에서 자신의 센터에 들어온 신청 목록을 확인한다.
2. 각 신청 건에 대해 수락 또는 거절을 선택한다.
3. 수락 시: 해당 의뢰 사업장이 센터의 창고 정보에 접근할 수 있도록 권한 관계가 수립된다.
4. 거절 시: 해당 의뢰가 거절 상태로 전환된다.

### 시나리오 G: 위탁 의뢰 취소 (의뢰자 측)

1. 의뢰를 보낸 사업장 담당자가 아직 처리되지 않은(신청 중) 의뢰를 취소할 수 있다.
2. 이미 수락되거나 거절된 의뢰는 취소 불가 (이미 처리된 건으로 간주).

## 6. 핵심 업무 규칙

**BR-1. 사업장명 필수**
- MUST: 사업장 정보 저장 시 사업장명은 반드시 입력해야 한다.
- NEVER: 빈 값으로 저장할 수 없다.

**BR-2. 센터명 필수 및 중복 금지**
- MUST: 물류센터 저장 시 센터명은 반드시 입력해야 한다.
- NEVER: 동일 사업장 내에서 같은 이름의 센터(사용 중인 것)는 중복 등록할 수 없다.
- 이유: 시나리오 B에서 저장 전 동일 사업장 내 동일 이름 센터 존재 여부를 검증한다.

**BR-3. 최소 1개 센터 사용 의무**
- NEVER: 사업장 내 모든 물류센터를 미사용 처리하거나 삭제하는 것은 허용되지 않는다.
- MUST: 저장 시점에 하나 이상의 센터가 사용 상태여야 한다.
- 이유: 시나리오 D에서 저장 완료 후 사용 중인 센터가 하나도 없으면 저장 자체가 차단된다.

**BR-4. 위탁 관계 존재 시 센터 조작 제한**
- NEVER: 위탁 대행 업체가 사용 중인 센터는 미사용으로 전환하거나 삭제할 수 없다.
- MUST: 위탁 관계를 먼저 해제해야 한다.
- 이유: 시나리오 C와 시나리오 D에서 위탁 대행 업체 사용 중 센터의 변경·삭제를 차단한다.

**BR-5. 사용자 권한 존재 시 센터 삭제 불가**
- NEVER: 해당 센터에 접근 권한이 부여된 일반 사용자(슈퍼 권한 제외)가 있으면 센터를 삭제할 수 없다.
- MUST: 사용자 권한을 먼저 정리해야 한다.
- 이유: 시나리오 D에서 사용자 권한 존재 시 센터 삭제를 차단한다.

**BR-6. 창고 존재 시 센터 삭제 불가**
- NEVER: 물류센터 내에 창고가 등록되어 있으면 센터를 삭제할 수 없다.
- MUST: 창고를 먼저 삭제해야 한다.
- 이유: 시나리오 D에서 창고 존재 시 센터 삭제를 차단한다.

**BR-7. 위탁 센터 주소·전화번호 필수**
- MUST: 물류 대행 센터로 지정(위탁 여부 = 사용)하려면 주소, 주소상세, 전화번호 세 항목을 모두 입력해야 한다.

**BR-8. 중복 의뢰 차단**
- NEVER: 이미 신청 중이거나 승인된 센터에는 중복으로 위탁 의뢰를 보낼 수 없다.
- MUST: 거절된 경우에만 재신청이 가능하다.
- 이유: 시나리오 E에서 신청중·승인 상태의 중복 신청을 차단하고 거절된 센터만 재신청 가능하다고 정의한다.

**BR-9. 취소는 신청 중 상태에서만 가능**
- MUST: 위탁 의뢰 취소는 대행 업체가 아직 처리하지 않은 신청 중 상태에서만 가능하다.
- NEVER: 이미 수락되거나 거절된 의뢰는 취소할 수 없다.
- 이유: 시나리오 G에서 신청 중 상태만 취소 가능하다고 정의한다.

**BR-10. 자동 창고 생성**
- MUST: 새 물류센터 등록 시 시스템이 사전 정의된 기본 창고 세트를 자동으로 생성한다.
- 이유: 시나리오 B에서 센터 등록과 동시에 기본 창고 세트 자동 생성을 정의한다.

**BR-11. 슈퍼 관리자 자동 권한 부여**
- MUST: 새 물류센터 등록 시 슈퍼 관리자 권한을 가진 사용자 전원에게 해당 센터 접근 권한이 자동으로 부여된다.
- 이유: 시나리오 B에서 센터 등록과 동시에 슈퍼 관리자 권한 사용자 전원에게 접근 권한을 자동 부여한다.

**BR-12. 수정 가능 사업장 범위**
- MUST: 사업장 권한 사용자는 자신이 속한 사업장만 수정할 수 있다.
- MUST: 센터 권한 사용자는 자신의 센터가 속한 사업장만 수정할 수 있다.
- NEVER: 화주 유형의 사업장은 이 화면에서 수정할 수 없다.
- 이유: 시나리오 A에서 권한 범위 내 수정 가능한 사업장 목록만 로드된다고 정의한다.

## 7. 권한별 업무 범위

| 권한 유형 | 접근 가능 사업장 | 특이사항 |
|---|---|---|
| 슈퍼 관리자 | 가입된 상위 사업장에 연결된 모든 사업장 | 새 센터 등록 시 자동으로 센터 권한 부여 |
| 사업장 권한 | 자신이 직접 속한 사업장만 | 사업자번호 기준으로 사업장 목록 필터링 |
| 센터 권한 | 자신의 센터가 속한 사업장 | 센터 소속 사업장만 목록에 노출 |
| 공통 제외 | 화주(납품업체) 유형 사업장 | 수정 화면 목록에서 제외 |
