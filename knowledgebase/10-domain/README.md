# 10-domain — 메뉴 횡단 공통 업무지식 (WHY)

여러 메뉴에 **공통으로 적용되는 업무 규칙·개념·엔티티 관계**를 둔다. 소스 코드를 읽어도 알 수 없는 "왜"만 기록한다.

## 무엇을 넣나
- 전사 공통 업무 규칙 (예: "재고는 음수가 될 수 없다", "확정 후에는 수정 불가")
- 핵심 도메인 용어 사전 (입고·출고·로케이션·이관 등)
- 엔티티 관계 (사업장 ↔ 물류센터 ↔ 거래처 등)

## 무엇을 넣지 않나
- **특정 메뉴 고유 업무지식** → `spec/{메뉴}/{메뉴}-00-domain.md`
- 코드 구조·아키텍처(HOW) → `patterns/`
- 소스에서 추출 가능한 현황 → 두지 않음 (소스가 정본, 위치는 `30-src-index`)

## 규칙
- **사람이 직접 작성**한다. 자동화 스킬(SD·KB)은 이 폴더를 생성·수정하지 않는다.
- 파일은 주제 단위로 나눈다. 예: `business-rules.md`, `glossary.md`, `entity-relations.md`
